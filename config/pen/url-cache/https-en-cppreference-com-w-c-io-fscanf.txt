<p>Reads data from a variety of sources, interprets it according to <code>format</code> and stores the results into given locations.
</p>
<div class="t-li1"><span class="t-li">1)</span> reads the data from <span class="t-lc"><a href="/w/c/io/std_streams" title="c/io/std streams">stdin</a></span></div>
<div class="t-li1"><span class="t-li">2)</span> reads the data from file stream <code>stream</code></div>
<div class="t-li1"><span class="t-li">3)</span> reads the data from null-terminated character string <code>buffer</code>. Reaching the end of the string is equivalent to reaching the end-of-file condition for <code>fscanf</code></div>
<div class="t-li1"><span class="t-li">4-6)</span> Same as <span class="t-v">(1-3)</span>, except that <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>c</span></span>, <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>s</span></span>, and <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span><span class="br0">&#91;</span></span></span> conversion specifiers each expect two arguments (the usual pointer and a value of type <span class="t-lc">rsize_t</span> indicating the size of the receiving array, which may be <span class="t-c"><span class="mw-geshi c source-c"><span class="nu0">1</span></span></span> when reading with a <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>c</span></span> into a single char) and except that the following errors are detected at runtime and call the currently installed <a href="/w/c/error/set_constraint_handler_s" title="c/error/set constraint handler s">constraint handler</a> function:
<dl><dd><ul><li> any of the arguments of pointer type is a null pointer
</li><li> <code>format</code>, <code>stream</code>, or <code>buffer</code> is a null pointer
</li><li> the number of characters that would be written by <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>c</span></span>, <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>s</span></span>, or <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span><span class="br0">&#91;</span></span></span>, plus the terminating null character, would exceed the second (<code>rsize_t</code>) argument provided for each of those conversion specifiers
</li><li> optionally, any other detectable error, such as unknown conversion specifier
</li></ul>
</dd><dd>As with all bounds-checked functions, <code>scanf_s</code> , <code>fscanf_s</code>, and <code>sscanf_s</code> are only guaranteed to be available if <span class="t-c"><span class="mw-geshi c source-c">__STDC_LIB_EXT1__</span></span> is defined by the implementation and if the user defines <span class="t-c"><span class="mw-geshi c source-c">__STDC_WANT_LIB_EXT1__</span></span> to the integer constant <span class="t-c"><span class="mw-geshi c source-c"><span class="nu0">1</span></span></span> before including <a href="/w/c/io" title="c/io"><tt>&lt;stdio.h&gt;</tt></a>.</div>
</dd></dl>

<p><br />
The <b>format</b> string consists of
</p>
<ul><li> non-whitespace multibyte characters except <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span></span></span>: each such character in the format string consumes exactly one identical character from the input stream, or causes the function to fail if the next character on the stream does not compare equal.
</li><li> whitespace characters: any single whitespace character in the format string consumes all available consecutive whitespace characters from the input (determined as if by calling <a href="/w/c/string/byte/isspace" title="c/string/byte/isspace"><tt>isspace</tt></a> in a loop). Note that there is no difference between <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;<span class="es1">\n</span>&quot;</span></span></span>, <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot; &quot;</span></span></span>, <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;<span class="es1">\t</span><span class="es1">\t</span>&quot;</span></span></span>, or other whitespace in the format string.
</li><li> conversion specifications. Each conversion specification has the following format:
</li></ul>
<dl><dd><ul><li> introductory <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span></span></span> character.
</li></ul>
</dd></dl>
<dl><dd><ul><li> <span class="t-mark">(optional)</span> assignment-suppressing character <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span></span></span>. If this option is present, the function does not assign the result of the conversion to any receiving argument.
</li></ul>
</dd></dl>
<dl><dd><ul><li> <span class="t-mark">(optional)</span> integer number (greater than zero) that specifies <i>maximum field width</i>, that is, the maximum number of characters that the function is allowed to consume when doing the conversion specified by the current conversion specification. Note that <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>s</span></span> and <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span><span class="br0">&#91;</span></span></span> may lead to buffer overflow if the width is not provided.
</li></ul>
</dd></dl>
<dl><dd><ul><li> <span class="t-mark">(optional)</span> <i>length modifier</i> that specifies the size of the receiving argument, that is, the actual destination type. This affects the conversion accuracy and overflow rules. The default destination type is different for each conversion type (see table below).
</li></ul>
</dd></dl>
<dl><dd><ul><li> conversion format specifier.
</li></ul>
</dd></dl>

</p><p>All conversion specifiers other than <span class="t-c"><span class="mw-geshi c source-c"><span class="br0">&#91;</span></span></span>, <span class="t-c"><span class="mw-geshi c source-c">c</span></span>, and <span class="t-c"><span class="mw-geshi c source-c">n</span></span> consume and discard all leading whitespace characters (determined as if by calling <a href="/w/c/string/byte/isspace" title="c/string/byte/isspace"><tt>isspace</tt></a>) before attempting to parse the input. These consumed characters do not count towards the specified maximum field width.
</p><p>The conversion specifiers <span class="t-c"><span class="mw-geshi c source-c">lc</span></span>, <span class="t-c"><span class="mw-geshi c source-c">ls</span></span>, and <span class="t-c"><span class="mw-geshi c source-c">l<span class="br0">&#91;</span></span></span> perform multibyte-to-wide character conversion as if by calling <a href="/w/c/string/multibyte/mbrtowc" title="c/string/multibyte/mbrtowc"><tt>mbrtowc</tt></a> with an <a href="/w/c/string/multibyte/mbstate_t" title="c/string/multibyte/mbstate t"><tt>mbstate_t</tt></a> object initialized to zero before the first character is converted.
</p><p>The conversion specifiers <span class="t-c"><span class="mw-geshi c source-c">s</span></span> and <span class="t-c"><span class="mw-geshi c source-c"><span class="br0">&#91;</span></span></span> always store the null terminator in addition to the matched characters. The size of the destination array must be at least one greater than the specified field width. The use of <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>s</span></span> or <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span><span class="br0">&#91;</span></span></span>, without specifying the destination array size, is as unsafe as <span class="t-lc"><a href="/w/c/io/gets" title="c/io/gets">gets</a></span>.
</p><p>The correct conversion specifications for the <a href="/w/c/types/integer" title="c/types/integer">fixed-width integer types</a> (<span class="t-lc"><a href="/w/c/types/integer" title="c/types/integer">int8_t</a></span>, etc) are defined in the header <a href="/w/c/types/integer" title="c/types/integer"><code>&lt;inttypes.h&gt;</code></a> (although <a href="/w/c/types/integer" title="c/types/integer"><tt>SCNdMAX</tt></a>, <a href="/w/c/types/integer" title="c/types/integer"><tt>SCNuMAX</tt></a>, etc is synonymous with <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>jd</span></span>, <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>ju</span></span>, etc).
</p><p>There is a <a href="/w/c/language/eval_order" title="c/language/eval order">sequence point</a> after the action of each conversion specifier; this permits storing multiple fields in the same "sink" variable.
</p><p>When parsing an incomplete floating-point value that ends in the exponent with no digits, such as parsing <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;100er&quot;</span></span></span> with the conversion specifier <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>f</span></span>, the sequence <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;100e&quot;</span></span></span> (the longest prefix of a possibly valid floating-point number) is consumed, resulting in a matching error (the consumed sequence cannot be converted to a floating-point number), with <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;r&quot;</span></span></span> remaining. Some existing implementations do not follow this rule and roll back to consume only <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;100&quot;</span></span></span>, leaving <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">&quot;er&quot;</span></span></span>, e.g. <a rel="nofollow" class="external text" href="https://sourceware.org/bugzilla/show_bug.cgi?id=1765">glibc bug 1765</a>.
</p><p>If a conversion specification is invalid, the behavior is undefined.
</p>
<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/io/fscanf&amp;action=edit&amp;section=2" title="Edit section: Return value">edit</a>]</span> <span class="mw-headline" id="Return_value">Return value</span></h3>
<div class="t-li1"><span class="t-li">1-3)</span> Number of receiving arguments successfully assigned (which may be zero in case a matching failure occurred before the first receiving argument was assigned), or <span class="t-lc"><a href="/w/c/io" title="c/io">EOF</a></span> if input failure occurs before the first receiving argument was assigned.</div>
<div class="t-li1"><span class="t-li">4-6)</span> Same as <span class="t-v">(1-3)</span>, except that <span class="t-lc"><a href="/w/c/io" title="c/io">EOF</a></span> is also returned if there is a runtime constraint violation.</div>
<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/io/fscanf&amp;action=edit&amp;section=3" title="Edit section: Complexity">edit</a>]</span> <span class="mw-headline" id="Complexity">Complexity</span></h3>
<p>Not guaranteed. Notably, some implementations of <code>sscanf</code> are <span class="texhtml" style="white-space: nowrap;">O(N)</span>, where <span class="t-c"><span class="mw-geshi c source-c">N <span class="sy1">=</span> <a href="http://en.cppreference.com/w/c/string/byte/strlen"><span class="kw558">strlen</span></a><span class="br0">&#40;</span>buffer<span class="br0">&#41;</span></span></span> <a rel="nofollow" class="external autonumber" href="https://sourceware.org/bugzilla/show_bug.cgi?id=17577">[1]</a>.
</p>
<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/io/fscanf&amp;action=edit&amp;section=4" title="Edit section: Notes">edit</a>]</span> <span class="mw-headline" id="Notes">Notes</span></h3>
<p>Because most conversion specifiers first consume all consecutive whitespace, code such as 
</p>
<div dir="ltr" class="mw-geshi" style="text-align: left;"><div class="c source-c"><pre class="de1">scanf<span class="br0">&#40;</span><span class="st0">&quot;%d&quot;</span>, <span class="sy3">&amp;</span>a<span class="br0">&#41;</span><span class="sy4">;</span>
scanf<span class="br0">&#40;</span><span class="st0">&quot;%d&quot;</span>, <span class="sy3">&amp;</span>b<span class="br0">&#41;</span><span class="sy4">;</span></pre></div></div>
<p>will read two integers that are entered on different lines (second <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>d</span></span> will consume the newline left over by the first) or on the same line, separated by spaces or tabs (second <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>d</span></span> will consume the spaces or tabs).
</p>
The conversion specifiers that do not consume leading whitespace, such as <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">%</span>c</span></span>, can be made to do so by using a whitespace character in the format string: <div dir="ltr" class="mw-geshi" style="text-align: left;"><div class="c source-c"><pre class="de1">scanf<span class="br0">&#40;</span><span class="st0">&quot;%d&quot;</span>, <span class="sy3">&amp;</span>a<span class="br0">&#41;</span><span class="sy4">;</span>
scanf<span class="br0">&#40;</span><span class="st0">&quot;&#160;%c&quot;</span>, <span class="sy3">&amp;</span>c<span class="br0">&#41;</span><span class="sy4">;</span> <span class="co1">// consume all consecutive whitespace after&#160;%d, then read a char</span></pre></div></div>
<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/io/fscanf&amp;action=edit&amp;section=5" title="Edit section: Example">edit</a>]</span> <span class="mw-headline" id="Example">Example</span></h3>
<div class="t-example"><div class="t-example-live-link"><div class="coliru-btn coliru-btn-run-init">Run this code</div></div>
<div dir="ltr" class="mw-geshi" style="text-align: left;"><div class="c source-c"><pre class="de1"><span class="co2">#define __STDC_WANT_LIB_EXT1__ 1</span>
<span class="co2">#include &lt;stdio.h&gt;</span>
<span class="co2">#include &lt;stddef.h&gt;</span>
<span class="co2">#include &lt;locale.h&gt;</span>
&#160;
<span class="kw4">int</span> main<span class="br0">&#40;</span><span class="kw4">void</span><span class="br0">&#41;</span>
<span class="br0">&#123;</span>
    <span class="kw4">int</span> i, j<span class="sy4">;</span>
    <span class="kw4">float</span> x, y<span class="sy4">;</span>
    <span class="kw4">char</span> str1<span class="br0">&#91;</span><span class="nu0">10</span><span class="br0">&#93;</span>, str2<span class="br0">&#91;</span><span class="nu0">4</span><span class="br0">&#93;</span><span class="sy4">;</span>
    <span class="kw4">wchar_t</span> warr<span class="br0">&#91;</span><span class="nu0">2</span><span class="br0">&#93;</span><span class="sy4">;</span>
    <a href="http://en.cppreference.com/w/c/locale/setlocale"><span class="kw898">setlocale</span></a><span class="br0">&#40;</span><a href="http://en.cppreference.com/w/c/locale/LC_categories"><span class="kw901">LC_ALL</span></a>, <span class="st0">&quot;en_US.utf8&quot;</span><span class="br0">&#41;</span><span class="sy4">;</span>
&#160;
    <span class="kw4">char</span> input<span class="br0">&#91;</span><span class="br0">&#93;</span> <span class="sy1">=</span> <span class="st0">&quot;25 54.32E-1 Thompson 56789 0123 56ß水&quot;</span><span class="sy4">;</span>
    <span class="coMULTI">/* parse as follows:
      &#160;%d: an integer
      &#160;%f: a floating-point value
      &#160;%9s: a string of at most 9 non-whitespace characters
      &#160;%2d: two-digit integer (digits 5 and 6)
      &#160;%f:  a floating-point value (digits 7, 8, 9)
      &#160;%*d: an integer which isn't stored anywhere
       ' ': all consecutive whitespace
      &#160;%3[0-9]: a string of at most 3 decimal digits (digits 5 and 6)
      &#160;%2lc: two wide characters, using multibyte to wide conversion  */</span>
    <span class="kw4">int</span> ret <span class="sy1">=</span> sscanf<span class="br0">&#40;</span>input, <span class="st0">&quot;%d%f%9s%2d%f%*d&#160;%3[0-9]%2lc&quot;</span>,
                     <span class="sy3">&amp;</span>i, <span class="sy3">&amp;</span>x, str1, <span class="sy3">&amp;</span>j, <span class="sy3">&amp;</span>y, str2, warr<span class="br0">&#41;</span><span class="sy4">;</span>
&#160;
    <a href="http://en.cppreference.com/w/c/io/fprintf"><span class="kw848">printf</span></a><span class="br0">&#40;</span><span class="st0">&quot;Converted&#160;%d fields:<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;i =&#160;%d<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;x =&#160;%f<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;str1 =&#160;%s<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;j =&#160;%d<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;y =&#160;%f<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;str2 =&#160;%s<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;warr[0] = U+%x<span class="es1">\n</span>&quot;</span>
           <span class="st0">&quot;warr[1] = U+%x<span class="es1">\n</span>&quot;</span>,
           ret, i, x, str1, j, y, str2, warr<span class="br0">&#91;</span><span class="nu0">0</span><span class="br0">&#93;</span>, warr<span class="br0">&#91;</span><span class="nu0">1</span><span class="br0">&#93;</span><span class="br0">&#41;</span><span class="sy4">;</span>
&#160;
<span class="co2">#ifdef __STDC_LIB_EXT1__</span>
    <span class="kw4">int</span> n <span class="sy1">=</span> sscanf_s<span class="br0">&#40;</span>input, <span class="st0">&quot;%d%f%s&quot;</span>, <span class="sy3">&amp;</span>i, <span class="sy3">&amp;</span>x, str1, <span class="br0">&#40;</span>rsize_t<span class="br0">&#41;</span><span class="kw4">sizeof</span> str1<span class="br0">&#41;</span><span class="sy4">;</span>
    <span class="co1">// writes 25 to i, 5.432 to x, the 9 bytes &quot;Thompson\0&quot; to str1, and 3 to n.</span>
<span class="co2">#endif</span>
<span class="br0">&#125;</span></pre></div></div>
