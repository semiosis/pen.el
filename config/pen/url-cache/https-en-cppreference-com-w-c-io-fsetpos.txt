<tr class="t-dcl">
<td class="t-dcl-nopad"> <div><span class="mw-geshi c source-c"><span class="kw4">int</span> fsetpos<span class="br0">&#40;</span> <a href="http://en.cppreference.com/w/c/io/FILE"><span class="kw883">FILE</span></a> <span class="sy2">*</span>stream, <span class="kw4">const</span> <a href="http://en.cppreference.com/w/c/io/fpos_t"><span class="kw884">fpos_t</span></a> <span class="sy2">*</span>pos <span class="br0">&#41;</span><span class="sy4">;</span></span></div></td>
<td class="t-dcl-nopad">  </td>
<td class="t-dcl-nopad">  </td>

<tr class="t-dcl-sep"><td></td><td></td><td></td></tr>

<p>Sets the file position indicator and the multibyte parsing state (if any) for the file stream <code>stream</code> according to the value pointed to by <code>pos</code>. 
</p><p>Besides establishing new parse state and position, a call to this function undoes the effects of <span class="t-lc"><a href="/w/c/io/ungetc" title="c/io/ungetc">ungetc</a></span> and clears the end-of-file state, if it is set.
</p><p>If a read or write error occurs, the error indicator (<span class="t-lc"><a href="/w/c/io/ferror" title="c/io/ferror">ferror</a></span>) for the stream is set.
</p>

<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/io/fsetpos&amp;action=edit&amp;section=3" title="Edit section: Notes">edit</a>]</span> <span class="mw-headline" id="Notes">Notes</span></h3>
<p>After seeking to a non-end position in a wide stream, the next call to any output function may render the remainder of the file undefined, e.g. by outputting a multibyte sequence of a different length.
