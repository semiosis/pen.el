The “How to Design Languages” Track&quot;" data-pltdoc="x">up</a>&nbsp;&nbsp;<a href="tue-aft-lab.html" title="forward to &quot;8 Lab Comfortable Macros&quot;" data-pltdoc="x">next &rarr;</a></span>&nbsp;</div><h3>7<tt>&nbsp;</tt><a name="(part._tue-aft-lecture)"></a>Lexical Scope, (Un)Hygienic Macros</h3><div class="SAuthorListBox"><span class="SAuthorList"><p class="author">Jay McCarthy</p></span></div><p><table cellspacing="0" cellpadding="0" class="boxed"><tr><td><p><span style="font-weight: bold">Goals</span></p></td></tr><tr><td><p> &#8212;<wbr></wbr> an understanding of scope&rsquo;s relation to macros</p></td></tr><tr><td><p> &#8212;<wbr></wbr> the macro "hygiene" system</p></td></tr><tr><td><p> &#8212;<wbr></wbr> how to break hygiene</p></td></tr><tr><td><p> &#8212;<wbr></wbr> how to use syntax parameters</p></td></tr><tr><td><p> &#8212;<wbr></wbr> macro-defining macros</p></td></tr></table></p><p>This afternoon we will build an involved macro to encode the class
pattern. The class pattern is commonly used in the non-pattern form in
class-oriented programming languages, like Java and C++. As a pattern,
however, it typically appears in languages like C or (original) JavaScript. The pattern is that
a function (called a constructor) returns an object that collects a
set of data together and defines methods over that set of data. These
methods can implicitly refer to the data (and each other), as well as
the object.</p><p>Rather than starting with the expanded code and deriving the macro
implementation, like we did previously, let&rsquo;s start with the ultimate
code we want to allow programmers to write. Then, we will work through
each of the components of the macro.</p><h4>7.1<tt>&nbsp;</tt><a name="(part._class1)"></a>An Example Class</h4><p>Our example class, <span class="RktSym">posn</span>, represents a point on the 2D
plane. We will be able to determine its distance from another point,
its distance from the origin, be able to slide it around in the X
dimension, and adjust it in the Y dimension. Let&rsquo;s first go over how
users of <span class="RktSym">posn</span>s will interact with them.</p><p><div class="SIntrapara">We will construct points by calling the function <span class="RktSym">posn</span> with
keywords for each of the fields. Here are a few examples:
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">posn</span><span class="hspace">&nbsp;</span><span class="RktPn">#:x</span><span class="hspace">&nbsp;</span><span class="RktVal">0</span><span class="hspace">&nbsp;</span><span class="RktPn">#:y</span><span class="hspace">&nbsp;</span><span class="RktVal">0</span><span class="RktPn">)</span></td></tr><tr><td><span class="RktPn">(</span><span class="RktSym">posn</span><span class="hspace">&nbsp;</span><span class="RktPn">#:y</span><span class="hspace">&nbsp;</span><span class="RktVal">4</span><span class="hspace">&nbsp;</span><span class="RktPn">#:x</span><span class="hspace">&nbsp;</span><span class="RktVal">2</span><span class="RktPn">)</span></td></tr><tr><td><span class="RktPn">(</span><span class="RktSym">posn</span><span class="hspace">&nbsp;</span><span class="RktPn">#:x</span><span class="hspace">&nbsp;</span><span class="RktVal">3</span><span class="hspace">&nbsp;</span><span class="RktPn">#:y</span><span class="hspace">&nbsp;</span><span class="RktVal">5</span><span class="RktPn">)</span></td></tr></table></blockquote></div></p><p><div class="SIntrapara">Given a point, we&rsquo;ll able to call methods by passing the object as the
first argument to method and providing the other arguments afterwards.
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span><span class="hspace">&nbsp;</span><span class="RktSym">p</span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym">posn</span><span class="hspace">&nbsp;</span><span class="RktPn">#:y</span><span class="hspace">&nbsp;</span><span class="RktVal">4</span><span class="hspace">&nbsp;</span><span class="RktPn">#:x</span><span class="hspace">&nbsp;</span><span class="RktVal">2</span><span class="RktPn">)</span><span class="RktPn">)</span></td></tr><tr><td><span class="RktPn">(</span><span class="RktSym">from-origin</span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym">p</span><span class="hspace">&nbsp;</span><span class="RktPn">. </span><span class="RktSym">slide</span><span class="RktPn"> .</span><span class="hspace">&nbsp;</span><span class="RktVal">1</span><span class="RktPn">)</span><span class="RktPn">)</span></td></tr><tr><td><span class="RktCmt">;</span><span class="RktCmt">&nbsp;</span><span class="RktCmt">=&gt; 5</span></td></tr></table></blockquote></div></p><p><div class="SIntrapara">Next, let&rsquo;s look at how the creator of the <span class="RktSym">posn</span> class will
make it. They will use the <span class="RktSym">classy</span> form:
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">classy</span><span class="hspace">&nbsp;</span><span class="RktSym">posn</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktSym">....more....</span><span class="RktPn">)</span></td></tr></table></blockquote></div></p><p><div class="SIntrapara">Inside of the body of <span class="RktSym">classy</span>, there can be a sequence of
<span class="RktSym"><a href="http://docs.racket-lang.org/reference/createclass.html#%28form._%28%28lib._racket%2Fprivate%2Fclass-internal..rkt%29._field%29%29" class="RktStxLink" data-pltdoc="x">field</a></span>s and <span class="RktSym">method</span>s. These can appear in any order,
although classy users of <span class="RktSym">classy</span> tend to put their field
definitions at the start.
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">classy</span><span class="hspace">&nbsp;</span><span class="RktSym">posn</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/createclass.html#%28form._%28%28lib._racket%2Fprivate%2Fclass-internal..rkt%29._field%29%29" class="RktStxLink" data-pltdoc="x">field</a></span><span class="hspace">&nbsp;</span><span class="RktSym">x</span><span class="RktPn">)</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/createclass.html#%28form._%28%28lib._racket%2Fprivate%2Fclass-internal..rkt%29._field%29%29" class="RktStxLink" data-pltdoc="x">field</a></span><span class="hspace">&nbsp;</span><span class="RktSym">y</span><span class="RktPn">)</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktSym">....more....</span><span class="RktPn">)</span></td></tr></table></blockquote></div></p><p><div class="SIntrapara">The definition of a method has implicit access to these fields and
implicitly has access to the object upon which the method is
invoked. For example, consider this poorly named function:
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">method</span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._magnitude%29%29" class="RktValLink" data-pltdoc="x">magnitude</a></span><span class="RktPn">)</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span><span class="hspace">&nbsp;</span><span class="RktSym">x</span><span class="hspace">&nbsp;</span><span class="RktSym">y</span><span class="RktPn">)</span><span class="RktPn">)</span></td></tr></table></blockquote></div><div class="SIntrapara">If we were to evaluate <span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._magnitude%29%29" class="RktValLink" data-pltdoc="x">magnitude</a></span><span class="stt"> </span><span class="RktPn">(</span><span class="RktSym">posn</span><span class="stt"> </span><span class="RktPn">#:y</span><span class="stt"> </span><span class="RktVal">4</span><span class="stt"> </span><span class="RktPn">#:x</span><span class="stt"> </span><span class="RktVal">2</span><span class="RktPn">)</span><span class="RktPn">)</span>, then
we would get <span class="RktVal">6</span>.</div></p><p><div class="SIntrapara">If a method needs to access the fields of another object (of the same
class), then it can do so using the <span class="RktSym">class.field</span> form. For
example, we could define the "distance from another point" method as:
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">method</span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym">distance-from</span><span class="hspace">&nbsp;</span><span class="RktSym">p2</span><span class="RktPn">)</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._sqrt%29%29" class="RktValLink" data-pltdoc="x">sqrt</a></span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28lib._racket%2Fmath..rkt%29._sqr%29%29" class="RktValLink" data-pltdoc="x">sqr</a></span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._-%29%29" class="RktPn">(</span><span class="RktSym">posn.x</span><span class="hspace">&nbsp;</span><span class="RktSym">p2</span><span class="RktPn">)</span><span class="RktPn">)</span><span class="RktPn">)</span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28lib._racket%2Fmath..rkt%29._sqr%29%29" class="RktValLink" data-pltdoc="x">sqr</a></span><span class="hspace">&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._-%29%29" the <span class="RktSym">posn</span>
class. So, a user could not write <span class="RktPn">(</span><span class="RktSym">posn.x</span><span class="stt"> </span><span class="RktPn">(</span><span class="RktSym">posn</span><span class="stt"> </span><span class="RktPn">#:x</span><span class="stt"> </span><span class="RktVal">3</span><span class="stt"> </span><span class="RktPn">#:y</span><span class="stt"> </span><span class="RktVal">5</span><span class="RktPn">)</span><span class="RktPn">)</span>
to get <span class="RktVal">3</span>. Similarly, the forms <span class="RktSym"><a href="http://docs.racket-lang.org/reference/createclass.html#%28form._%28%28lib._racket%2Fprivate%2Fclass-internal..rkt%29._field%29%29" class="RktStxLink" data-pltdoc="x">field</a></span> and
<span class="RktSym">method</span> are illegal outside of the <span class="RktSym">classy</span> macro.</p><p><div class="SIntrapara">The <span class="RktSym">classy</span> macro provides another special form,
<span class="RktSym">update</span>, for constructing a new object, based on the current
object, with a slightly different field. We could use this to define
the slide and adjust functions:

will simply take an expression and bind the variable <span class="RktSym">it</span> to

<span class="RktSym"><a href="http://docs.racket-lang.org/reference/stx-patterns.html#%28form._%28%28lib._racket%2Fprivate%2Fstxcase-scheme..rkt%29._with-syntax%29%29" class="RktStxLink" data-pltdoc="x">with-syntax</a></span> and <span class="RktPn">#:with</span>) but rather than the

possible in Racket to introduce new names and manually break the
default hygiene rules. However, it is considered poor taste to
introduce new names like this, based solely on manufactured names.</p><p>The more beautiful way of doing this is to use <span style="font-style: italic">syntax

definition of the macro that binds them and the macro specifically

provided as an argument to the macro. (In other words, it can
<span class="RktSym"><a href="http://docs.racket-lang.org/reference/let.html#%28form._%28%28lib._racket%2Fprivate%2Fletstx-scheme..rkt%29._let%29%29" class="RktStxLink" data-pltdoc="x">let</a></span> the name, but it cannot <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> the name.)</p><p>An example use comparable to the last use is as follows:</p><blockquote class="SCodeFlow"><table cellspacing="0" cellpadding="0" class="RktBlk"><tr><td><span class="RktPn">(</span><span class="RktSym">with-it</span><span class="hspace">&nbsp;</span><span class="RktVal">5</span></td></tr><tr><td><span class="hspace">&nbsp;&nbsp;</span><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span><span class="hspace">&nbsp;</span><span class="RktSym">it</span><span class="hspace">&nbsp;</span><span class="RktSym">it</span><span class="RktPn">)</span><span class="RktPn">)</span></td></tr><tr><td><span class="RktCmt">;</span><span class="RktCmt">&nbsp;</span><span class="RktCmt">=&gt; 10</span></td></tr></table></blockquote><p>However, if we consider a situation where the user attempts to
reference <span class="RktSym">it</span> outside of a context where the macro bound it,
