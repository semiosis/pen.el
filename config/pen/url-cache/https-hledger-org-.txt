<h1 id="hledger"><a class="header" href="#hledger">hledger</a></h1>
<div id="tagline">
Fast, friendly, robust<br> <a href="https://plaintextaccounting.org">plain text accounting</a> software
<!-- <br>‚ö°Ô∏èüí™üèº‚ù§Ô∏è -->
</div>
<h2 id="hledger-is"><a class="header" href="#hledger-is">hledger is...</a></h2>
<ul>
<li>free GPL-licensed accounting software that runs on unix, mac, windows, and the web</li>
<li>based on readable, version-controllable, future-proof plain text files</li>
<li>good for tracking money, investments, time, or any countable commodity</li>
<li>good for learning  more about <a href="accounting.html">double entry accounting</a></li>
<li>shipped with <a href="web.html">web</a>, <a href="ui.html">terminal</a>, <a href="basics.html">command line</a>, <a href="https://hledger.org/hledger-web.md#json-api">JSON</a> and <a href="https://hackage.haskell.org/package/hledger-lib">Haskell</a> interfaces</li>
<li>the most user friendly <a href="https://plaintextaccounting.org">plain text accounting app</a></li>
<li>comparable to <a href="ledger.html">Ledger</a> and <a href="beancount.html">Beancount</a></li>
<li>built with the powerful <a href="https://haskell.org">Haskell language</a></li>
<li>fast, lightweight, reliable (we pay $50 for <a href="REGRESSIONS.html">regression reports or fixes</a>)</li>
<li>free from vendor lock-in</li>
<li>local, not cloud-based; your private financial data remains under your control</li>
<li><a href="scripting.html">scripting-friendly</a> and automatable</li>
<li>comfortable for technical users, but also usable by others. Knowing about 
<a href="https://tutorial.djangogirls.org/en/intro_to_command_line/">command lines</a>, 
<a href="https://www.codementor.io/@kmcgillivray/introduction-to-text-editors-and-the-command-line-j0upqatas">text editors</a> and 
<a href="https://betterexplained.com/articles/a-visual-guide-to-version-control/">version control</a> 
is helpful, but not required (much)</li>
<li>good at importing CSV from banks and other financial institutions</li>
<li>actively developed and supported by <a href="http://joyful.com">Simon Michael</a> and <a href="CREDITS.html">contributors</a> since 2007</li>
<li>strengthened by your <a href="support.html">feedback</a>, <a href="CONTRIBUTING.html">work</a> and <a href="sponsor.html">funding</a>. üôèüèª</li>
</ul>

<h2 id="quick-start"><a class="header" href="#quick-start">Quick start</a></h2>
<p>Welcome! This plain text accounting stuff is useful and more fun than it sounds - care to give it a try ?</p>
<p><a href="install.html">Install</a>, then see <a href="start.html">Get Started</a>, or the Examples below,
or run <code>hledger</code> to see help and demos.
Full documentation is ready when you need it, in the sidebar to the left.
(If not visible, click/tap the horizontal-lines icon at top left.)</p>

<h2 id="examples"><a class="header" href="#examples">Examples</a></h2>
<p>Here are three transactions in <a href="hledger.html#journal">journal format</a>,
recorded in the <a href="hledger.html#input">journal file</a> (<code>$LEDGER_FILE</code> or <code>~/.hledger.journal</code>)
by <a href="hledger.html#add"><code>hledger add</code></a> or <a href="create-a-journal.html">other method</a>.
The <a href="hledger.html#account-names">account names</a> and <a href="hledger.html#amounts">amounts</a> are separated by at least two spaces;
a positive amount means &quot;added to this account&quot;, negative means &quot;removed from this account&quot;.
hledger will check that each transaction's amounts sum to zero; one of them may be omitted for convenience.</p>
<pre><code class="language-journal">
2023-01-01 opening balances            ; &lt;- First transaction sets starting balances.
    assets:bank:checking        $1000  ; &lt;- Account names can be anything.
    assets:bank:savings         $2000  ; &lt;- Colons indicate subaccounts.
    assets:cash                  $100  ; &lt;- 2+ spaces are required before the amount.
    liabilities:credit card      $-50  ; &lt;- A debt; these are negative.
    equity:opening/closing     $-3050  ; &lt;- Starting balances come from equity.
                                       ;    Equity is also usually negative.
                                       ;    (Reports can show as positive when needed.)

2023-02-01 GOODWORKS CORP              ; &lt;- Date order is recommended but optional.
    assets:bank:checking       $1000
    income:salary                      ; &lt;- $-1000 is inferred here to balance the txn.
                                       ;    Income amounts are negative.

2023-02-15 market
    expenses:food             $50
    assets:cash                        ; &lt;- $-50 is inferred here.
</code></pre>
<p>You can run reports like so:</p>
<pre><code class="language-cli">$ hledger bs
Balance Sheet 2023-02-15

                         || 2023-02-15 
=========================++============
 Assets                  ||            
-------------------------++------------
 assets:bank:checking    ||      $2000 
 assets:bank:savings     ||      $2000 
 assets:cash             ||        $50 
-------------------------++------------
                         ||      $4050 
=========================++============
 Liabilities             ||            
-------------------------++------------
 liabilities:credit card ||        $50 
-------------------------++------------
                         ||        $50 
=========================++============
 Net:                    ||      $4000 
</code></pre>
<pre><code class="language-cli">$ hledger is -MTA
Income Statement 2023-01-01..2023-02-28

               || Jan    Feb    Total  Average 
===============++==============================
 Revenues      ||                              
---------------++------------------------------
 income:salary ||   0  $1000    $1000     $500 
---------------++------------------------------
               ||   0  $1000    $1000     $500 
===============++==============================
 Expenses      ||                              
---------------++------------------------------
 expenses:food ||   0    $50      $50      $25 
---------------++------------------------------
               ||   0    $50      $50      $25 
===============++==============================
 Net:          ||   0   $950     $950     $475 
</code></pre>
<pre><code class="language-cli">$ hledger aregister checking
Transactions in assets:bank:checking and subaccounts:
2023-01-01 opening balances     as:ba:savings, as:..         $1000         $1000
2023-02-01 GOODWORKS CORP       in:salary                    $1000         $2000
</code></pre>
<h3 id="declarations"><a class="header" href="#declarations">Declarations</a></h3>
<p>If you use other account names, it's useful to declare their <a href="hledger.html#account-types">account types</a>:</p>
<pre><code class="language-journal">
account actifs                          ; type:Asset, 2+ spaces required before the ;
account actifs:banque:compte courant    ; type:Cash
account actifs:banque:compte d'√©pargne  ; type:Cash
account actifs:esp√®ces                  ; type:Cash
account passifs                         ; type:Liability
account capitaux propres                ; type:Equity
account revenus                         ; type:Revenue
account d√©penses                        ; type:Expense
</code></pre>
<p>Or declare all accounts, currencies and tags, if you want <a href="hledger.html#strict-mode">strict error checking</a>:</p>
<pre><code class="language-journal">
account assets                   ; type:A, 2+ spaces required before the ;
account assets:bank              ; type:C
account assets:bank:checking
account assets:bank:savings
account assets:cash              ; type:C
account liabilities              ; type:L
account liabilities:credit card
account equity                   ; type:E
account equity:opening/closing
account income                   ; type:R
account income:salary
account income:gifts
account expenses                 ; type:X
account expenses:rent
account expenses:food
account expenses:gifts

commodity $1000.00

tag type
</code></pre>
<pre><code class="language-cli">$ hledger check --strict
$ 
</code></pre>
<p>Declaring accounts also helps set their preferred <a href="hledger.html#account-display-order">display order</a>:</p>
<pre><code class="language-cli">$ hledger accounts -t
assets
  bank
    checking
    savings
  cash
liabilities
  credit card
equity
  opening/closing
income
  salary
  gifts
expenses
  rent
  food
  gifts
</code></pre>
<p>You can declare <a href="hledger.html#alias-directive">account aliases</a> to save typing:</p>
<pre><code class="language-journal">
alias chk  = assets:bank:checking
alias cash = assets:cash
alias card = liabilities:creditcard
alias food = expenses:food

...

2023-02-15 market
    food          $50
    cash
</code></pre>

<h3 id="time-tracking"><a class="header" href="#time-tracking">Time tracking</a></h3>
<p>hledger can also read time logs in <a href="hledger.html#timeclock">timeclock</a> format:</p>
<pre><code class="language-timeclock">
i 2023/03/27 09:00:00 projects:a
o 2023/03/27 17:00:34
i 2023/03/31 22:21:45 personal:reading:online
o 2023/04/01 02:00:34
</code></pre>
<pre><code>$ hledger -f 2023.timeclock register -D
2023-03-27   projects:a                         8.01h         8.01h
2023-03-31   personal:reading:online            1.64h         9.65h
2023-04-01   personal:reading:online            2.01h        11.66h
</code></pre>
<p>Or in <a href="hledger.html#timedot">timedot</a> format:</p>
<pre><code class="language-timedot">
2023/2/1
biz:research  .... ..
fos:hledger   .... .... ....

2023/2/2
fos:ledger    0.25
fos:haskell   .5
biz:client1   .... ....
</code></pre>
<pre><code>$ hledger -f 2023.timedot balance -tDTA  # tree, Daily, Total, Average
Balance changes in 2023-02-01..2023-02-02:

            || 2023-02-01  2023-02-02    Total  Average 
============++==========================================
 biz        ||       1.50        2.00     3.50     1.75 
   client1  ||          0        2.00     2.00     1.00 
   research ||       1.50           0     1.50     0.75 
 fos        ||       3.00        0.75     3.75     1.88 
   haskell  ||          0        0.50     0.50     0.25 
   hledger  ||       3.00           0     3.00     1.50 
   ledger   ||          0        0.25     0.25     0.12 
------------++------------------------------------------
            ||       4.50        2.75     7.25     3.62 
</code></pre>
<h3 id="csv-import"><a class="header" href="#csv-import">CSV import</a></h3>
<p>hledger can read <a href="hledger.html#csv">CSV</a> (or SSV, TSV, or other character-separated) files representing transactions:</p>
<pre><code class="language-csv">
&quot;Date&quot;,&quot;Notes&quot;,&quot;Amount&quot;
&quot;2023/2/22&quot;,&quot;DEPOSIT&quot;,&quot;50.00&quot;
&quot;2023/2/23&quot;,&quot;TRANSFER TO SAVINGS&quot;,&quot;-10.00&quot;
</code></pre>
<pre><code class="language-rules">
# bank.csv.rules  # this rules file tells hledger how to read bank.csv
skip 1
fields date, description, amount
currency $
account1 assets:bank:checking

if WHOLE FOODS
 account2 expenses:food

if (TO|FROM) SAVINGS
 account2 assets:bank:savings
</code></pre>
<pre><code class="language-cli">$ hledger -f bank.csv print
2023-02-22 DEPOSIT
    assets:bank:checking          $50.00
    income:unknown               $-50.00

2023-02-23 TRANSFER TO SAVINGS
    assets:bank:checking         $-10.00
    assets:bank:savings           $10.00

</code></pre>
<p>The <a href="hledger.html#import">import</a> command detects and adds just new transactions to the journal (works with most CSVs):</p>
<pre><code class="language-cli">$ hledger import bank.csv
imported 2 new transactions from bank.csv
</code></pre>
<pre><code class="language-cli">$ hledger import bank.csv
no new transactions found in bank.csv
</code></pre>
<pre><code class="language-cli">$ hledger aregister checking
2023-01-01 opening balances     as:ba:savings, as:..      $1000.00      $1000.00
2023-02-01 GOODWORKS CORP       in:salary                 $1000.00      $2000.00
2023-02-22 DEPOSIT              in:unknown                  $50.00      $2050.00
2023-02-23 TRANSFER TO SAVINGS  as:ba:savings              $-10.00      $2040.00
</code></pre>
