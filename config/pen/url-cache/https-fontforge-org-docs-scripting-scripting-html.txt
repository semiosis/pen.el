<h1>Scripting FontForge<a class="headerlink" href="#scripting-fontforge" title="Permalink to this heading">¶</a></h1>
<p>FontForge includes two interpreters so you can write scripts to modify fonts.
One of these interpreters is <a class="reference internal" href="python.html"><span class="doc">python</span></a>, one is a legacy language I
came up with. FontForge may be configured with either or both of these. If
configured with both then fontforge will make a guess at which to use based on
the script file’s extension (“py” means use the python interpreter, “ff” or “pe”
means use the old interpreter)</p>

<section id="invoking-scripts">
<span id="scripting-starting"></span><h2>Invoking scripts<a class="headerlink" href="#invoking-scripts" title="Permalink to this heading">¶</a></h2>
<p>If you start fontforge with a script on the command line it will not put up any
windows and it will exit when the script is done. The script can be in a file,
or just a string presented as an argument. You may need to specify which
interpreter to use with the -lang argument.</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span>$ fontforge -script scriptfile.pe {arguments}
$ fontforge -c &quot;script-string&quot; {arguments}
$ fontforge -lang={ff|py} -c &quot;script-string&quot;
</pre></div>
</div>
<p>FontForge can also be used as an interpreter to which the shell will
automatically pass scripts. If you a mark your script files as executable</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span>$ chmod +x scriptfile.pe
</pre></div>
</div>
<p>and begin each one with the line</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="ch">#!/usr/local/bin/fontforge</span>
</pre></div>
</div>
<p>(or wherever fontforge happens to reside on your system) then you can invoke the
script just by typing</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span>$ scriptfile.pe {fontnames}
</pre></div>
</div>
<p>If you wish FontForge to read a script from stdin then you can use “-” as a
“filename” for stdin. (If you build FontForge without X11 then fontforge will
attempt to read a script file from <code class="docutils literal notranslate"><span class="pre">stdin</span></code> if none is given on the command
line.)</p>
<p>You can also start a script from within FontForge with
<span class="menuselection">File ‣ Execute Script</span>, and you can use the Preference Dlg to
define a set of frequently used scripts which can be invoked directly by menu.</p>
<p>The scripting language provides access to much of the functionality found in the
font view’s menus. It does not currently (and probably never will) provide
access to everything. (If you find a lack let me know, I may put it in for you).
It does not provide commands for building up a glyph out of splines, instead it
allows you to do high level modifications to glyphs.</p>
<p>If you set the environment variable <code class="docutils literal notranslate"><span class="pre">FONTFORGE_VERBOSE</span></code> (it doesn’t need a
value, just needs to be set) then FontForge will print scripts to stdout as it
executes them.</p>
<p>You may set the environment variable <code class="docutils literal notranslate"><span class="pre">FONTFORGE_LANGUAGE</span></code> to either “py” (for
python) or “ff” or “pe” (for native scripting) as another way to determne what
interpreter to use.</p>
</section>
<section id="scripting-language">
<span id="id1"></span><h2>Scripting Language<a class="headerlink" href="#scripting-language" title="Permalink to this heading">¶</a></h2>
<div class="admonition note">
<p class="admonition-title">Note</p>
<p>This section covers FontForge’s native scripting functionality.</p>
<p>For the Python scripting functionality, see <a class="reference internal" href="python.html"><span class="doc">here</span></a>.</p>
</div>
<p>The syntax is rather like a mixture of C and shell commands. Every file
corresponds to a procedure. As in a shell script arguments passed to the file
are identified as $1, $2, … $n. $0 is the file name itself. $argc gives the
number of arguments. $argv[&lt;expr&gt;] provides array access to the arguments.</p>
<p>Terms can be</p>
<ul>
<li><p>A variable name (like “$1” or “i” or “&#64;fontvar” or “_global”)</p>
<p>The scope of the variable depends on the initial character of its name.</p>
<ul class="simple">
<li><p>A ‘$’ signifies that it is a built-in variable. The user cannot create any new
variables beginning with ‘$’. Some, but not all, of these may be assigned to.</p></li>
<li><p>A ‘_’ signifies that the variable is global, it is always available. You can use
these to store context across different script files (or to access data within
nested script files).</p></li>
<li><p>A ‘&#64;’ signifies that the variable is associated with the font. Any two scripts
looking at the same font will have access to the same variables.</p></li>
<li><p>A variable which begins with a letter is a local variable. It is only meaningful
within the current script file. Nested script files may have different variables
with the same names.</p></li>
</ul>
</li>
<li><p>an integer expressed in decimal, hex or octal</p></li>
<li><p>a unicode code point (which has a prefix of “0u” or “0U” and is followed by a
string of hex digits. This is only used by the select command.</p></li>
<li><p>a real number (in “C” locale format – “.” as the decimal point character)</p></li>
<li><p>A string which may be enclosed in either double or single quotes. String tokens
are limited to 256 bytes. “n” can be used to represent a newline character. (If
you need longer strings use the concatenation operator).</p></li>
<li><p>a procedure to call or file to invoke.</p></li>
<li><p>an expression within parentheses</p></li>
<li><p>a series of expressions (separated by commas) within brackets, as
<code class="docutils literal notranslate"><span class="pre">[1,</span> <span class="pre">2,</span> <span class="pre">3,</span> <span class="pre">5,</span> <span class="pre">8]</span></code>, used to create an array.</p></li>
</ul>
<p>There are three different comments supported:</p>
<ul class="simple">
<li><p>Starting with a “#” character and proceeding to end of line</p></li>
<li><p>Starting with “//” and proceeding to end of line</p></li>
<li><p>Starting with “/<em>” and proceeding to “</em>/”</p></li>
</ul>
<section id="expressions">
<span id="scripting-expressions"></span><h3>Expressions<a class="headerlink" href="#expressions" title="Permalink to this heading">¶</a></h3>
<p>Expressions are similar to those in C, a few operators have been omitted, a few
added from shell scripts. Operator precedence has been simplified slightly. So
operators (and their precedences) are:</p>
<ul>
<li><p>unary operators (+, -, !, ~, ++ (prefix and postfix), –(prefix and postfix), ()
(procedure call), [] (array index), :h, :t, :r, :e</p>
<p>Most of these are as expected in C, the last four are borrowed from shell
scripts and are applied to strings</p>
<ul class="simple">
<li><p>:h gives the head (directory) of a pathspec</p></li>
<li><p>:t gives the tail (filename) of a pathspec</p></li>
<li><p>:r gives the pathspec without the extension (if any)</p></li>
<li><p>:e gives the extension</p></li>
</ul>
</li>
<li><p>*, /, % (binary multiplicative operators)</p></li>
<li><p>+, - (binary arithmetic operators)</p>
<p>If the first operand of + is a string then + will be treated as concatenation
rather than addition. If the second operand is a number it will be converted to
a string (decimal representation) and then concatenated. If the first operand of
+ is an array then + will do array concatenation – if the second argument is
also an array the two will be concatenated ([1,2] + [3,4] yields [1,2,3,4],
while [1,2] + 3 yields [1,2,3]). Otherwise these are the normal arithmetric
operations.</p>
</li>
<li><p>==, !=, &gt;, &lt;, &gt;=, &lt;= (comparison operators, may be applied to either two
integers or two strings)</p></li>
<li><p>&amp;&amp;, &amp; (logical and, bitwise and. (logical and will do short circuit evaluation))</p></li>
<li><p>||, |, ^ (logical or, bitwise or, bitwise exclusive or (logical or will do short
circuit evaluation))</p></li>
<li><p>=, +=, -=, *=, /=, %= (assignment operators as in C. The += will act as
concatenation if the first operand is a string.)</p></li>
</ul>
<p>Note there is no comma operator, and no “?:” operator. The precedence of “and”
and “or” has been simplified, as has that of the assignment operators.</p>
<p>Procedure calls may be applied either to a name token, or to a string. If the
name or string is recognized as one of FontForge’s internal procedures it will
be executed, otherwise it will be assumed to be a filename containing another
fontforge script file, this file will be invoked (since filenames can contain
characters not legal in name tokens it is important to allow general strings to
specify filenames). If the procedure name does not contain a directory then it
is assumed to be in the same directory as the current script file. As a
convenience, it is not necessary to include the script’s extension if it is
either “.ff” or “.pe”. At most 25 arguments can be passed to a procedure.</p>
<p>Arrays are passed by reference, strings and integers are passed by value.</p>
<p>Variables may be created by assigning a value to them (only with the “=”), so:</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">i</span><span class="o">=</span><span class="mi">3</span>
</pre></div>
</div>
<p>could be used to define “i” as a variable. Variables are limited in scope to the
current file, they will not be inherited by called procedures.</p>
<p>A statement may be</p>
<ul>
<li><p>an expression</p></li>
<li><div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">if</span> <span class="p">(</span> <span class="n">expression</span> <span class="p">)</span>
    <span class="n">statements</span>
<span class="p">{</span><span class="n">elseif</span> <span class="p">(</span> <span class="n">expression</span> <span class="p">)</span>
    <span class="n">statements</span><span class="p">}</span>
<span class="p">[</span><span class="k">else</span>
    <span class="n">statements</span><span class="p">]</span>
<span class="n">endif</span>
</pre></div>
</div>
</li>
<li><div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="k">while</span> <span class="p">(</span> <span class="n">expression</span> <span class="p">)</span>
    <span class="n">statements</span>
<span class="n">endloop</span>
</pre></div>
</div>
</li>
<li><div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">foreach</span>
    <span class="n">statements</span>
<span class="n">endloop</span>
</pre></div>
</div>
</li>
<li><p><code class="docutils literal notranslate"><span class="pre">break</span></code></p></li>
<li><p><code class="docutils literal notranslate"><span class="pre">return</span> <span class="pre">[</span> <span class="pre">expression</span> <span class="pre">]</span></code></p></li>
<li><p><code class="docutils literal notranslate"><span class="pre">shift</span></code></p></li>
</ul>
<p>As with C, non-zero expressions are defined to be true.</p>
<p>A return statement may be followed by a return value (the expression) or a
procedure may return nothing (void).</p>
<p>The shift statement is stolen from shell scripts and shifts all arguments down
by one. (argument 0, the name of the script file, remains unchanged.</p>
<p>The foreach statement requires that there be a current font. It executes the
statements once for each glyph in the selection. Within the statements only one
glyph at a time will be selected. After execution the selection will be restored
to what it was initially. (Caveat: Don’t reencode the font within a foreach
statement).</p>
<p>Statements are terminated either by a new line (you can break up long lines with
backslash newline) or a semicolon.</p>
<p>Trivial example:</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span><span class="n">i</span><span class="o">=</span><span class="mi">0</span><span class="p">;</span>   <span class="c1">#semicolon is not needed here, but it&#39;s ok</span>
<span class="k">while</span> <span class="p">(</span> <span class="n">i</span><span class="o">&lt;</span><span class="mi">3</span> <span class="p">)</span>
   <span class="k">if</span> <span class="p">(</span> <span class="n">i</span><span class="o">==</span><span class="mi">1</span> <span class="o">/*</span> <span class="n">pointless</span> <span class="n">comment</span> <span class="o">*/</span> <span class="p">)</span>
        <span class="n">Print</span><span class="p">(</span> <span class="s2">&quot;Got to one&quot;</span> <span class="p">)</span>   <span class="o">//</span> <span class="n">Another</span> <span class="n">comment</span>
   <span class="n">endif</span>
   <span class="o">++</span><span class="n">i</span>
<span class="n">endloop</span>
</pre></div>
</div>
<p>FontForge maintains the concept of a “current font”– almost all commands refer
only to the current font (and require that there be a font). If you start a
script with <span class="menuselection">File ‣ Execute Script</span>, the font you were editing
will be current, otherwise there will be no initial current font. The Open(),
New() and Close() commands all change the current font. FontForge also maintains
a list of all fonts that are currently open. This list is in no particular
order. The list starts with $firstfont.</p>
<p>Similarly when working with cid keyed fonts, FontForge works in the “current sub
font”, and most commands refer to this font. The CIDChangeSubFont() command can
alter that.</p>
</section>

<li><p><code class="docutils literal notranslate"><span class="pre">$&lt;Preference</span> <span class="pre">Item&gt;</span></code> (for example <code class="docutils literal notranslate"><span class="pre">$AutoHint</span></code>) allows you to examine the
value of that preference item (to set it use
<a class="reference internal" href="scripting-alpha.html#SetPref" title="SetPref"><code class="xref ff ff-func docutils literal notranslate"><span class="pre">SetPref()</span></code></a>)</p></li>
</ul>
<p>The following example will perform an action on all loaded fonts:</p>
<div class="highlight-default notranslate"><div class="highlight"><pre><span></span>file = $firstfont
while ( file != &quot;&quot; )
   Open(file)
   /* Do Stuff */
   file = $nextfont
endloop
</pre></div>
</div>

<div class="highlight-default notranslate"><div class="highlight"><pre><span></span>#!/usr/local/bin/fontforge
#This is the sfddiff script which compares two fonts

if ( Strtol($version) &lt; 20060330 )
  Error( &quot;Please upgrade to a more recent version of fontforge&quot; )
endif

flags=0x789
outfile=&quot;&quot;

while ( $argc &gt; 1 &amp;&amp; Strsub($1,0,1)==&quot;-&quot; )
  temp = $1
  if ( Strsub(temp,1,2)==&#39;-&#39; )
    temp = Strsub(temp,1)
  endif

  if ( temp==&quot;-ignorehints&quot; )
    flags = flags &amp; ~0x8
  elseif ( temp==&quot;-ignorenames&quot; )
    flags = flags &amp; ~0x100
  elseif ( temp==&quot;-ignoregpos&quot; )
    flags = flags &amp; ~0x200
  elseif ( temp==&quot;-ignoregsub&quot; )
    flags = flags &amp; ~0x400
  elseif ( temp==&quot;-ignorebitmaps&quot; )
    flags = flags &amp; ~0x80
  elseif ( temp==&quot;-exact&quot; )
    flags = flags | 0x2
  elseif ( temp==&quot;-warn&quot; )
    flags = flags | 0x44
  elseif ( temp==&quot;-merge&quot; )
    flags = flags | 0x1800
    shift
    outfile = $1
  elseif ( temp==&quot;-help&quot; )
    Print( &quot;sfddiff: [--version] [--help] [--usage] [--ignorehints] [--ignorenames] [--ignoregpos] [--ignoregsup] [--ignorebitmaps] [--warn] [--exact] fontfile1 fontfile2&quot; )
    Print( &quot; Compares two fontfiles&quot; )
    Print( &quot; --ignorehints: Do not compare postscript hints or truetype instructions&quot; )
    Print( &quot; --ignorenames: Do not compare font names&quot; )
    Print( &quot; --ignoregpos:  Do not compare kerning, etc.&quot; )
    Print( &quot; --ignoregsub:  Do not compare ligatures, etc.&quot; )
    Print( &quot; --ignorebitmaps: Do not compare bitmap strikes&quot; )
    Print( &quot; --exact:       Normally sfddiff will match contours which are not exact&quot; )
    Print( &quot;                but where the differences are slight (so you could compare&quot; )
    Print( &quot;                truetype and postscript and get reasonable answers). Also&quot; )
    Print( &quot;                normally sfddiff will unlink references before it compares&quot; )
    Print( &quot;                (so you can compare a postscript font (with no references)&quot; )
    Print( &quot;                to the original source (which does have references)). Setting&quot;)
    Print( &quot;                this flag means glyphs must match exactly.&quot;)
    Print( &quot; --warn:        Provides a warning when an exact match is not found&quot; )
    Print( &quot; --merge outfile: Put any outline differences in the backgrounds of&quot; )
    Print( &quot;                appropriate glyphs&quot; )
return(0)
  elseif ( temp==&quot;-version&quot; )
    Print( &quot;Version 1.0&quot; )
return(0)
  else
break
  endif
  shift
endloop

if ( $argc!=3 || $1==&quot;--usage&quot; || $1==&quot;-usage&quot; )
  Print( &quot;sfddiff: [--version] [--help] [--usage] [--ignorehints] [--ignorenames] [--ignoregpos] [--ignoregsup] [--ignorebitmaps] [--warn] [--exact] [--merge outfile] fontfile1 fontfile2&quot; )
return(0)
endif

Open($2)
Open($1)
CompareFonts($2,&quot;-&quot;,flags)
if ( outfile!=&quot;&quot; )
  Save(outfile)
endif

<section id="the-execute-script-dialog">
<span id="scripting-execute"></span><h2>The Execute Script dialog<a class="headerlink" href="#the-execute-script-dialog" title="Permalink to this heading">¶</a></h2>
<p>This dialog allows you to type a script directly in to FontForge and then run
it. Of course the most common case is that you’ll have a script file somewhere
that you want to execute, so there’s a button [Call] down at the bottom of the
dlg. Pressing [Call] will bring up a file picker dlg looking for files with the
extension *.pe (you can change that by typing a wildcard sequence and pressing
the [Filter] button). After you have selected your script the appropriate text
to text to invoke it will be placed in the text area.</p>
<p>The current font of the script will be set to whatever font you invoked it from
(in python this is <a class="reference internal" href="python/fontforge.html#fontforge.activeFont" title="fontforge.activeFont"><code class="xref py py-func docutils literal notranslate"><span class="pre">fontforge.activeFont()</span></code></a>).</p>
<p>Note that if you try to print from a script the output will go to stdout. If you
have invoked fontforge from a window manager’s menu stdout will often be bound
to /dev/null and not useful. Try starting fontforge from the command line
instead.</p>
</section>
<section id="the-scripts-menu">
<span id="scripting-menu"></span><h2>The Scripts Menu<a class="headerlink" href="#the-scripts-menu" title="Permalink to this heading">¶</a></h2>
<p>You can use the preference dialog to create a list of frequently used scripts.
Invoke <a class="reference internal" href="../ui/dialogs/prefs.html#prefs-scripts"><span class="std std-ref">File-&gt;Preferences</span></a> and select the Scripts tag. In
this dialog are ten possible entries, each one should have a name (to be
displayed in the menu) and an associated script file to be run.</p>
<p>After you have set up your preferences you can invoke scripts from the font
view, either directly from the menu
(<span class="menuselection">File ‣ Scripts ‣ &lt;your name&gt;</span>) or by a hot key. The first
script you added will be invoked by Cnt-Alt-1, then second by Cnt-Alt-2, and the
tenth by Cnt-Alt-0.</p>
<p>The current font of the script will be set to whatever font you invoked it from.</p>
</section>
