                  <span class="commtext c00">&gt; GPT-3 struggles with large numbers, decimal numbers, and negative numbers. When used it returns answers that are close but often incorrect.<p>Regarding GPT-3&#x27;s &quot;guesstimates,&quot; intuitively it feels like the network <i>has</i> to guess because it hasn&#x27;t been given a way to do exact computation--a neural network is built out of nonlinear functions--even if it &quot;understands&quot; the prompt (for whatever value you want to give to &quot;understand&quot;).<p>Are there any techniques that involve giving the model access to an oracle and allowing it to control it?  To continue the analogy, this would be the equivalent of giving GPT-3 a desk calculator.<p>If this is a thing, I have other questions.  How do you train against it?  Would the oracle have to be differentiable?  (There are multiple ways to operate a desk calculator to evaluate the same expression.)  Also, what control interface would the model need so that it can learn to use the oracle?  (Would GPT-3 emit a sequence of 1-hot vectors that represent functions to do, and would the calculator have &quot;registers&quot; that can be fed directly from the input text?  Some way of indirectly referring to operands so the model doesn&#x27;t have to lossily handle them.)</span>

                  <span class="commtext c00">There are many papers trying to couple language models with external modules.<p>In the Retrieval-Enhanced Transformer (RETRO) paper a large language model was coupled with a similarity based text index. It can populate the prompt with relevant information from the index thus being more grounded and update-able.<p>In another paper (AlphaCode) the language model was coupled with a compiler and could run programs and check if they match the expected outputs for a few test cases. The model was able to solve competition style coding problems above average human score.<p>In another paper (Language Models as Zero Shot Planners) a language model generates commands to navigate a virtual home environment and performs tasks. The knowledge in the LM helps in quickly learning tasks.<p>A recent one can learn new concepts by simple conversation, then apply them where necessary. You can talk-train your model. (Memory assisted prompt editing to improve GPT 3 after deployment)<p>So the trend is to add &quot;toys&quot; on language models - a simulator, a compiler, a search engine, a long term memory module.<p>I&#x27;d like to see a recursive language model, that can sub-call itself to decompose problems.</span>
