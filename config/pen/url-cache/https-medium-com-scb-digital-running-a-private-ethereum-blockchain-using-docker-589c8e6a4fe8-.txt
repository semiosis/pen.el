id="ed64" class="pw-post-title iu iv iw bo ix iy iz ja jb jc jd je jf jg jh ji jj jk jl jm jn jo jp jq jr js fy">Running a Private Ethereum Blockchain using Docker</h1></div><div class=""><h2 id="b902" class="pw-subtitle-paragraph jt iv iw bo b ju jv jw jx jy jz ka kb kc kd ke kf kg kh ki kj kk br">This post describes how to setup and run a cluster of nodes to build a private Ethereum network in Docker environment.</h2></div><p id="fd4a" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">Although there are many public Ethereum test networks for testing or using for smart contract and decentralized application (dApp) developments, running our own private Ethereum network could still be useful as well as helping us to gain a better understanding on how blockchain works.</p><h1 id="5d7b" class="lh li iw bo lj lk ll lm ln lo lp lq lr kc ls kd lt kf lu kg lv ki lw kj lx ly fy">The private Ethereum network nodes</h1><p id="015e" class="pw-post-body-paragraph kl km iw kn b ko lz jx kq kr ma ka kt ku mb kw kx ky mc la lb lc md le lf lg ip fy">Ethereum network is a peer-to-peer network consists of multiple nodes running the Ethereum client such as <a class="au me" href="https://geth.ethereum.org/" rel="noopener ugc nofollow" target="_blank">Geth</a> or <a class="au me" href="https://github.com/openethereum/openethereum" rel="noopener ugc nofollow" target="_blank">OpenEthereum</a>.</p><p id="b564" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">In this post, we are setting up a private network with 3 nodes as follows:</p><ol class=""><li id="0103" class="mf mg iw kn b ko kp kr ks ku mh ky mi lc mj lg mk ml mm mn fy"><strong class="kn ix">Bootnode</strong> — the bootstrap node that is used for peer discovery purpose. It listens on port <em class="mo">30303</em>, the other nodes joining the network connect to this bootnode first.</li><li id="e9be" class="mf mg iw kn b ko mp kr mq ku mr ky ms lc mt lg mk ml mm mn fy"><strong class="kn ix">JSON-RPC endpoint </strong>— this node exposes JSON-RPC API over HTTP endpoint on port <em class="mo">8545</em>. We will publish the port <em class="mo">8545</em> of this node container to the host machine to allow external interaction with this private blockchain.</li><li id="8c79" class="mf mg iw kn b ko mp kr mq ku mr ky ms lc mt lg mk ml mm mn fy"><strong class="kn ix">Miner</strong> — this node is responsible for mining (the process of creating a new block in our blockchain). When the miner node successfully mines a new block, it receives the rewards into the configured account.</li></ol><figure class="mv mw mx my gv mz gj gk paragraph-image"><div class="gj gk mu"><img alt="Client nodes in the Private Ethereum Network" class="ea na nb" src="https://miro.medium.com/max/802/1*w11EC-gEOidmlRKRqh6liw.png" width="401" height="211"/></div><figcaption class="nc ec gl gj gk nd ne bo b bp bq br">Figure 1. Client nodes in the Private Ethereum Network</figcaption></figure><h1 id="9224" class="lh li iw bo lj lk ll lm ln lo lp lq lr kc ls kd lt kf lu kg lv ki lw kj lx ly fy">The genesis block</h1><p id="18e0" class="pw-post-body-paragraph kl km iw kn b ko lz jx kq kr ma ka kt ku mb kw kx ky mc la lb lc md le lf lg ip fy">To setup a private Ethereum blockchain, the Ethereum client requires some information for creating the first block which is called the genesis block.</p><p id="1e72" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">Here is a simple custom <em class="mo">genesis.json</em> containing information to create the genesis block.</p><figure class="mv mw mx my gv mz"><div class="m l ct"><div class="nf ng l"></div></div></figure><p id="0d6d" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">We will not go into detail of each parameter here but let’s consider these 2</p><ul class=""><li id="eb29" class="mf mg iw kn b ko kp kr ks ku mh ky mi lc mj lg nh ml mm mn fy"><strong class="kn ix">config.chainId</strong> — this is the identifier to tell the nodes which blockchain they are on. We set it to be <code class="fn ni nj nk nl b">1214</code> to avoid conflicting with the public Ethereum networks.</li><li id="b534" class="mf mg iw kn b ko mp kr mq ku mr ky ms lc mt lg nh ml mm mn fy"><strong class="kn ix">config.ethash</strong> — this indicates that our blockchain will be using <em class="mo">proof-of-work</em> as the consensus engine.</li></ul><h1 id="96bd" class="lh li iw bo lj lk ll lm ln lo lp lq lr kc ls kd lt kf lu kg lv ki lw kj lx ly fy">Docker image for Ethereum client</h1><p id="f3f5" class="pw-post-body-paragraph kl km iw kn b ko lz jx kq kr ma ka kt ku mb kw kx ky mc la lb lc md le lf lg ip fy">For the Ethereum nodes in our private blockchain, we will use Go Ethereum (Geth) as the client. So let’s create a <em class="mo">Dockerfile</em> for building the image of our Ethereum client.</p><ol class=""><li id="8470" class="mf mg iw kn b ko kp kr ks ku mh ky mi lc mj lg mk ml mm mn fy">We use <a class="au me" href="https://hub.docker.com/r/ethereum/client-go/" rel="noopener ugc nofollow" target="_blank">official Geth image</a> from Docker Hub the base image.</li></ol><pre class="mv mw mx my gv nm eg nn"><span id="9367" class="fy no li iw nl b cr np nq l nr">FROM ethereum/client-go:v1.10.1</span></pre><p id="66cd" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">2. Copy the <em class="mo">genesis.json</em> into the image and use it to initialize the genesis block</p><pre class="mv mw mx my gv nm eg nn"><span id="d492" class="fy no li iw nl b cr np nq l nr">COPY genesis.json /tmp<br/>RUN geth init /tmp/genesis.json</span></pre><p id="c7d5" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">3. Remove the <em class="mo">nodekey</em> file</p><pre class="mv mw mx my gv nm eg nn"><span id="0787" class="fy no li iw nl b cr np nq l nr">RUN rm -f ~/.ethereum/geth/nodekey</span></pre><p id="c859" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">The <em class="mo">nodekey</em> file is generated when Geth is initializing, it is created the file in the folder named <em class="mo">geth</em> under its data directory which (by default) is located at <em class="mo">~/.ethereum</em>.</p><p id="1128" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">The <em class="mo">nodekey</em> file is used to generate the Geth <em class="mo">enode</em> which is kind of the id of each node in Ethereum network.</p><p id="660f" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">As we will use the same image built from the <em class="mo">Dockerfile</em> to run all the nodes in our Ethereum network, we have to delete the <em class="mo">nodekey</em> file created while initializing Geth. So the file will be recreated with different key when each node is starting, otherwise every node in our private network will have the same <em class="mo">enode</em> which will prevent the nodes to be connected and synced together.</p><p id="5db4" class="pw-post-body-paragraph kl km iw kn b ko kp jx kq kr ks ka kt ku kv kw kx ky kz la lb lc ld le lf lg ip fy">4.
