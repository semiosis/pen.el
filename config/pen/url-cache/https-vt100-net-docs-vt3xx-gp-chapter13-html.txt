<h1 id="S13"><span class="chapnum">13</span> 4010/4014 Mode</h1>
<p>The VT300 can support industry-standard Tektronix 4010/4014 software packages.
This chapter describes how to select and use 4010/4014 mode. The chapter
assumes you have a working knowledge of Tektronix 4010/4014 capabilities.</p>
<p>The 4010/4014 terminals use direct-view storage tube technology. The VT300
uses raster-scan technology. The 4010/4014 mode supports those features that
can migrate to a raster terminal.</p>
<p class="note">NOTE: Tektronix software may run differently on the VT300 than on other
terminals, due to differences in terminal design.</p>
<h2 id="S13.1">Supported Features</h2>
<p>The 4010/4014 mode supports the following modes and functions. This chapter
has a section on each feature.</p>
<table class="dpadded">
<tbody>
<tr>
<td>Alpha mode</td>
<td>Processes text characters.</td>
</tr>
<tr>
<td>Graph mode</td>
<td>Processes vectors from endpoints defined by absolute coordinates.</td>
</tr>
<tr>
<td>Incremental plot mode</td>
<td>Similar to graph mode, but plots points relative to the current cursor position.</td>
</tr>
<tr>
<td>Point plot mode</td>
<td>Similar to graph mode, but does not draw vectors. Only plots the points specified by absolute coordinates.</td>
</tr>
<tr>
<td>Graphics input (GIN) mode</td>
<td>A local mode, similar to report position interactive in ReGIS. You can use the keyboard, a mouse, or graphics tablet to move the cursor and send position reports to the host.</td>
</tr>
<tr>
<td>Strap options</td>
<td>Supported as set-up mode options. See the Graphics Set-Up screen in <cite>Installing and Using the VT330/VT340 Video Terminal</cite>.</td>
</tr>
<tr>
<td>Control characters</td>
<td>Supports some control characters to control terminal actions in 4010/4014 mode.</td>
</tr>
<tr>
<td>Escape sequences</td>
<td>Supports some escape sequences to control 4010/4014 functions only. The way escape sequences work in 4010/4014 mode is independent of the way they work in text mode.</td>
</tr>
<tr>
<td>Bypass condition</td>
<td>Prevents the terminal from responding to data sent by the terminal and echoed by the host.</td>
</tr>
</tbody>
</table>
<h2 id="S13.2">Restrictions</h2>
<p>The VT300 cannot support some 4010/4014 features, due to the differences between
direct-view storage tube and raster-scan display technologies. The following
sections describe these limitations.</p>
<h3 id="S13.2.1">Write-Through Mode</h3>
<p>With 4010/4014 terminals, you can draw images and characters on the tube
without storing them. The application must refresh these images to keep them
visible. The intensity level of the images depends on the refresh rate.</p>
<p>The VT300 can simulate write-through functions by using raster writing modes.
The "<a href="#S13.8">Escape Sequences</a>" section in this chapter describes the capabilities and
limitations of raster writing modes.</p>
<h3 id="S13.2.2">Character Sizes</h3>
<p>In 4010/4014 mode, the VT300 uses one of two character modes, aligned or
enlarged.</p>
<table class="dpadded">
<tbody>
<tr>
<td>Aligned mode</td>
<td>The terminal uses four character sizes. The characters in all four sizes are small, but conform to the Tektronix terminal.</td>
</tr>
<tr>
<td>Enlarged mode</td>
<td>The terminal uses two character sizes. The enlarged characters are larger and easier to read than the smaller aligned characters.</td>
</tr>
</tbody>
</table>
<p>In enlarged mode, Tektronix software that relies on strict registration of characters
to pixels creates character distortion on the VT300. This problem does
not occur with aligned characters. Although the smaller aligned characters are
more difficult to read, they are not subject to pixel distortion.</p>
<p class="note">NOTE: You can select the aligned or enlarged characters from the Graphics
Set-Up screen.</p>
<h3 id="S13.2.3">Fonts</h3>
<p>Loadable fonts and alternate hard fonts are not available in 4010/4014 mode.</p>
<h3 id="S13.2.4">Enhanced Graphics Module (EGM)</h3>
<p>The Tektronix enhanced graphics module (EGM) is a 4014 option that provides
a number of special features. The VT300 supports a number of features available
through the EGM option. See "<a href="#S13.14">Graph Mode</a>" in this chapter.</p>
<h2 id="S13.3">Screen Addressing</h2>
<p>The 4010/4014 series terminals use Tekpoints as their unit of screen addressing.
By default, there is a 1024 × 768 Tekpoint matrix. Graphics input (GIN)
mode uses this default matrix. Other operating modes use the 12-bit addressing
capability of the EGM option, increasing the visible matrix to a 4096 × 3072
array.</p>
<p>By contrast, the VT300 has an 800 × 480 pixel matrix. In 4010/4014 mode,
the VT300 uses a 623 × 480 pixel array that is centered on the screen. In
vector drawing, the VT300 uses a 614 × 460 pixel array. The extra space is
used for character drawing.</p>
<p>Because the VT300 has a lower pixel resolution, several Tekpoints map to one
pixel. The VT300 maps Tekpoints to the nearest pixel in the 623 × 480 array.</p>
<h2 id="S13.4">Set-Up Support</h2>
<p>The 4010/4014 terminals have four strap options you can select by moving
jumper wires on the terminal's circuit cards.</p>
<ul class="col2 bare">
<li>CR effect</li>
<li>LF effect</li>
<li>Graphics input terminators</li>
<li>DEL implies LO Y</li>
</ul>
<p>On the VT300, you can select these strap options from the Graphics Set-Up
screen. See <cite>Installing and Using the VT330/VT340 Video Terminal</cite> for details.</p>

<p>The function of a valid ASCII character may depend on (1) which 4010/4014
operating mode the terminal is using when the character is received, and (2)
whether or not the ASCII character is part of an escape sequence.</p>
<p><a href="#T13-1">Table 13-1</a> lists the valid ASCII codes for 4010/4014 mode and briefly describes
their different functions. Later sections on control characters, escape sequences,
and 4010/4014 operating modes provide more details.</p>

<h2 id="S13.8">Escape Sequences</h2>
<p>The following sections describe the escape sequences you can use in 4010/4014
mode. The terminal ignores any other escape sequences received in this mode.
The valid escape sequences can come from the host or the VT300 keyboard.</p>
<p>The sequences contain control characters. To send the control character codes
from the keyboard, you hold down the <b>Ctrl</b> key and press another key. <a href="#T13-3">Table
13-3</a> lists the keys to press for each control character.</p>
<p>In GIN mode, the terminal buffers and stores escape sequences, until the terminal
exits GIN mode. After exiting GIN mode, the terminal performs the buffered
escape sequences.</p>
<h3 id="S13.8.1">Available Functions</h3>
<p>You can perform the following functions with escape sequences in 4010/4014
mode.</p>
<ul>
<li>Request terminal status.</li>
<li>Print hard copy of the bitmap.</li>
<li>Set bypass condition.</li>
<li>Set alpha mode.</li>
<li>Set GIN mode.</li>
<li>Set point plot mode.</li>
<li>Select raster writing mode features.</li>
<li>Select character sizes.</li>
<li>Select vector patterns.</li>
<li>Prevent response to carriage returns (CRs) or line feeds (LFs).</li>
<li>Set LCE flag. (Indicates an escape sequence introduction condition.)</li>
<li>Delete character.</li>
</ul>

<h3 id="S13.13.2">Margins for Two-Column Writing</h3>
<p>In alpha mode, you can use two-column writing. This form of writing uses two
margins. Margin 1 is at the left edge of the display area. Margin 2 is at the
center of each row in the display area.</p>
<p>Margin 1 is active when the terminal writes rows of characters from the left
edge. Margin 2 is active when the terminal writes from the center of the display
area.</p>
<p>The active margin automatically switches after one of the following events.</p>
<ul>
<li><p>The terminal fills the last row for the currently active margin.</p></li>
<li><p>The terminal receives a line feed on the last row of the display.</p></li>
</ul>
<p>The terminal then wraps characters around to the top row of the display, at
the new margin.</p>
<p>Since 4010/4014 terminals are storage tube terminals, they cannot scroll. In
4010/4014 mode, you cannot scroll. In alpha mode, the VT300 writes characters
as follows. (This description assumes that the terminal does not receive any
control characters while writing.)</p>
<ol>
<li><p>Character processing starts on the top row, from the upper-left corner to the upper-right corner.</p></li>
<li><p>When the terminal reaches the right edge of a row, the terminal wraps the next character to the left edge of next row down.</p></li>
<li><p>The terminal continues writing until it fills the bottom row.</p></li>
<li><p>When the bottom row is full, the next character wraps around to the top row at the middle of the screen.</p></li>
<li><p>The terminal now writes characters from the middle of the screen to the right edge, overstriking any characters already displayed.</p></li>
<li><p>As each row fills, the next character wraps to the middle of the next row.</p></li>
<li><p>The terminal continues writing until it fills the last row.</p></li>
<li><p>When the last row is full, the next character wraps around to the top row at the left margin. Then the process starts again.</p></li>
</ol>
<p>You can use one-column or two-column writing. One-column writing uses the
full width of the screen. If you want one-column writing, then you must clear
the screen before characters wrap around to margin 2.</p>
<p>If you want two-column writing, then you should insert CR and LF in each row
before writing reaches margin 2. (You can insert CR alone if you set <b>401X CR
Processing</b> to "CR" in the Graphics Set-Up screen.) This step prevents overstriking
of characters.</p>

<h3 id="S13.13.4">Erasing Characters with Spaces and Backspaces</h3>
<p>In alpha mode, the VT300 erases a character when the terminal receives a
space (SP) character immediately after a backspace (BS) character. The space
character erases any character in the current character cell.</p>
<p class="note">NOTE: This operation is not consistent with Tektronix 4010 or 4014 terminals.</p>
<p>This action is similar to using the &lt;x] key for correcting typing errors. Most
operating systems send a backspace, space, and backspace when they receive
the delete (DEL) character.</p>
<p>If the space character follows any character other than a backspace, the VT300
does not erase a character. Therefore, you can use the space character for
positioning.</p>
<h2 id="S13.14">Graph Mode</h2>
<p>In graph mode, the terminal draws vectors between the absolute coordinate values
you select. The absolute coordinates are Tekpoint values, mapped to the
nearest corresponding pixel on the VT300 screen. The terminal draws the vectors
in the currently selected line pattern. The next section describes the line
patterns available.</p>
<p>The 4014 with the enhanced graphics module (EGM) has a 4096 × 4096
square matrix. The top 25 percent of the Y addresses are above the top of the
display area. If you specify coordinates in this top area, they are tracked accurately.
However, the terminal only draws the part of the requested vector that
appears in the display area. The rest of the vector is clipped.</p>
<p>In graph mode, the VT300 recognizes any valid 4010/4014 mode escape sequences
or control characters.</p>

<h3 id="S13.14.2">Drawing Command</h3>
<p>You use the GS control character to enter graph mode from alpha mode. In
graph mode, GS defines the start of a vector.</p>
<p>GS does not draw vectors from the cursor position. GS draws the vector from
the first coordinate value specified to the next value specified. If you specify
more than two coordinates after a GS control character, the terminal draws a
vector between every two coordinates. In the following example, the lowercase
letters represent coordinate specifiers.</p>
<p class="example">GS a b c d</p>
<p>The terminal draws vectors from point <b>a</b> to point <b>b</b>, from point <b>b</b> to point <b>c</b>,
and from point <b>c</b> to point <b>d</b>. However, if you use the following command</p>
<p class="example">GS a b GS c d</p>
<p>the terminal draws two separate vectors, one from <b>a</b> to <b>b</b>, and another from <b>c</b>
to <b>d</b>.</p>
<h3 id="S13.14.3">Encoding Coordinates</h3>
<p>In 4010/4014 mode, you can use 10-bit or 12-bit addressing. You use 10-bit
addressing when the Tekpoint matrix of the screen is defined as 1024 × 768.
You use the 12-bit mode when the Tekpoint matrix of the screen is defined as
4096 × 3072.</p>
<p>In either case, the VT300 screen remains defined as a 623 × 480 pixel array.
The Tekpoints are always mapped to the nearest corresponding pixel. In graph
mode, the screen is a 614 × 480 pixel array. The extra space is for character
drawing.</p>
<p>For 10-bit addressing, coordinates are encoded into 4 bytes. For 12-bit addressing,
coordinates are encoded into 5 bytes. <a href="#T13-4">Table 13-4</a> shows the order used to
send these bytes and identifies their formats. The terminal does not send the
extra byte for 10-bit addressing, but the order of the remaining bytes is the same.</p>

<h2 id="S13.15">Point Plot Mode</h2>
<p>In point plot mode, the terminal draws single pixels instead of vectors.
Remember, 4010/4014 mode uses Tekpoints for screen addressing. Tekpoints
are expressed as absolute coordinate values. The terminal turns on a pixel that
most closely corresponds to the Tekpoint coordinate values.</p>
<p>Point plot mode uses the same 10-bit and 12-bit addressing methods as graph
mode. See the previous "<a href="#S13.14.3">Encoding Coordinates</a>" section for details. As with
graph mode, you can specify a coordinate that is not in the actual display area.
The terminal tracks these points, but they do not appear on the screen.</p>
<p>You can enter point plot mode from alpha or graph mode, using the FS control
character. Point plot mode functions do not use any other control characters or
escape sequences. However, the terminal recognizes most valid 4010/4014 mode
control characters and escape sequences in point plot mode.</p>
<h2 id="S13.16">Incremental Plot Mode</h2>
<p>In this mode, the terminal plots points relative to the current cursor position.
The screen addressing area is 4096 × 3072 Tekpoints. Since each pixel equals
several Tekpoints, you may have to send several characters to move the drawing
point to a new pixel.</p>
<p>You can enter incremental mode from all modes (except GIN mode) by using
the RS control character or ESC RS sequence. When you select RS, the terminal
uses the current cursor position for relative movement.</p>
<p>Incremental plot mode always draws with a solid line at normal intensity. You
can enter commands to alter the pattern register or intensity within
incremental plot mode, but they do not affect the drawing until you leave
incremental plot mode.</p>
<p>In incremental plot mode, you can change the cursor position without drawing.
A space turns the electron beam off. Then you can use other characters to
move the cursor in different directions. The P character turns the beam back

<h2 id="S13.17">Graphics Input (GIN) Mode</h2>
<p>You can select GIN mode by using the ESC SUB sequence. When the terminal
enters GIN mode, the following events occur.</p>
<ul>
<li><p>The VT300 crosshair cursor appears, with the hairs intersecting at the active position.</p>
<ul>
<li><p>If you are using the arrow keys or a mouse to move the cursor, the initial cursor position is at the center of the screen.</p></li>
<li><p>If you are using a graphics tablet, the initial cursor position is at the position specified by the tablet. You cannot use the arrow keys to move the cursor when you use a graphics tablet.</p></li>
</ul></li>
<li><p>The bypass condition is on.</p></li>
<li><p>The terminal buffers characters received from the host. The terminal
does not process the characters until you leave GIN mode.</p>
<p>When the terminal's input buffer is nearly full, the terminal sends
an XOFF character to the host. XOFF tells the host to stop sending
characters. If the host ignores the XOFF, the terminal loses any
characters received when the input buffer is full.</p></li>
</ul>

<h3 id="S13.17.2">Using a Locator Device (Mouse or Tablet)</h3>
<p>In addition to the arrow keys, you can use a mouse or graphics tablet to move
the cursor in GIN mode. <a href="chapter15.html">Chapter 15</a> describes how to use a locator device with
the VT300.</p>
<p>If you try to move the crosshair cursor past a screen boundary, the cursor
stops at the boundary.</p>
<p class="note">NOTE: Applications cannot cause the terminal to exit GIN mode.</p>
<p>You must leave GIN mode from the keyboard, as follows.</p>
<ol>
<li><p>Move the cursor to the desired position.</p></li>
<li><p>Press any active key (except the arrow keys) on the keyboard. The
key you press must be active in VT100 mode. When you press the
key, the following events occur.</p>
<ul>
<li><p>The terminal sends the character code or control function of the key to the host.</p></li>
<li><p>The terminal sends the current coordinates of the crosshair cursor to the host. These coordinates are in 10-bit addressing format. (The terminal never sends the extra bytes to the host.)</p></li>
<li><p>The crosshair cursor disappears from the screen.</p></li>
<li><p>The VT300 leaves GIN mode and enters alpha mode.</p></li>
</ul></li>
<li><p>At this point, the VT300 is still in the bypass condition. You can
turn this condition off by using one of the control characters and
escape sequences listed in the "<a href="#S13.12">Bypass Condition</a>" section in this chapter.</p></li>
</ol>
