						<h1 class="entry-title">Cron alarm clock</h1>
										<div class="comments-link">
					<a href="https://linuxtidbits.wordpress.com/2008/01/19/cron-alarm-clock/#comments">8 Replies</a>				</div><!-- .comments-link -->

				<div class="entry-content">
			<p><a href="http://openclipart.org/detail/31153/alarm-clock-by-rg1024"><img alt="" src="https://i0.wp.com/openclipart.org/image/150px/svg_to_png/31153/alarm_clock.png" class="alignright" width="112" height="150" /></a></p>
<p>Surprisingly I found out that there are a lack of alarm clocks in Linux.  I&#8217;ve tried a few but really was impressed with none.  So I decided to go with what the Linux uses for some of it&#8217;s timed events: Cron.  Cron can be a good alarm and its easy to use.</p>
<p>The program <code>crontab</code> is used to create and edit cron jobs and the cron daemon checks for cronjobs a the top of every minute.  A command line mp3 player can be used here but I like use Audacious.  Crong will need to be told what X display to play Audacious on to discover the current display:</p>
<pre style="overflow:auto;width:auto;border:solid #A4DEA4;border-width:.1em .1em .1em .6em;padding:.2em .4em .2em .6em;background-color:#E8EEE8;font-size:1em;white-space:pre;">env | grep DISPLAY</pre>
<p>Then as regular user edit crontab:</p>
<pre style="overflow:auto;width:auto;border:solid #A4DEA4;border-width:.1em .1em .1em .6em;padding:.2em .4em .2em .6em;background-color:#E8EEE8;font-size:1em;white-space:pre;">crontab -e</pre>
<p>Here&#8217;s the cronjob layout representation:</p>
<pre style="overflow:auto;width:auto;border:solid #C2C2C2;border-width:.1em;padding:.2em .6em;background-color:#E6E6ED;font-size:1em;white-space:pre;">#   minute (0-59),
#   |   hour (0-23),
#   |   |   day of the month (1-31),
#   |   |   |   month of the year (1-12),
#   |   |   |   |   day of the week (0-6 with 0=Sunday).
#   |   |   |   |   |   commands</pre>
<p>Enter the time (multiple times, days need to be separated with commas) and the script to execute for the alarm.  The asterik can be used (*) to satisfy all variables.  Here&#8217;s mine for audacious:</p>
<pre style="overflow:auto;width:auto;border:solid #C2C2C2;border-width:.1em;padding:.2em .6em;background-color:#E6E6ED;font-size:1em;white-space:pre;">07  21  *   *   1,2,3,4,5   env DISPLAY=:0.0 audacious /home/user/My\ Music/Other/Alarms/301gq.mp3</pre>
<p>I found some good alarm tracks at <a href="http://www.policeinterceptor.com/navysounds.htm">this site</a> the were perfect for the task.</p>
<p>To list the crontab do:</p>
<pre>crontab -l </pre>
<p></p>

<h3>Other Options</h3>
<h4>Wmtimer</h4>
<p>Wmtimer works well for Gnome users it leaves a small window on the screen which can&#8217;t be hidden but it is able use the system beep or execute a command on an alarm/timer event.   Starting it from the command line is pretty easy:</p>
<pre style="overflow:auto;width:auto;border:solid #A4DEA4;border-width:.1em .1em .1em .6em;padding:.2em .4em .2em .6em;background-color:#E8EEE8;font-size:1em;white-space:pre;">wmtimer -a -t 18:39:00 -e "mpg321 ~/Music/Other/Alarms/301gq.wav"</pre>
<h4>Audacious</h4>
<p>Audacious also has alarm ability but it&#8217;s buried DeeeEEep in it&#8217;s preferences ( Preferences &gt; Plugins &gt; General ).</p>
<h4>Alarm Clock</h4>
<p>Thats actually its name.  Despite the unexciting name, <a href="http://alarm-clock.54.pl/">Alarm Clock</a> is a very nice addition.  Its a python script for the Gnome desktop that has the ability to run an application, display a message, and to use as a tray icon.  Its a good application and easy to install.</p>
<p><a href='https://linuxtidbits.files.wordpress.com/2008/02/alarm-clock.png' title='alarm-clock.png'><img src='https://linuxtidbits.files.wordpress.com/2008/02/alarm-clock.png?w=625' alt='alarm-clock.png' /></a></p>

		const id = 'wp-enqueue-dynamic-script:' + handle + ':' + type + ':' + (index + 1);
		const template = document.getElementById(id);
		if (!template) {
			return Promise.reject();
		}

		const script = document.createElement( 'script' );
		script.innerHTML = template.innerHTML;
		document.body.appendChild( script );
		return Promise.resolve();
	}

	function loadExternalScript(handle) {
		if (!urls[handle]) {
			return Promise.resolve();
		}

		return fetches[handle].then(() => {
			return new Promise((resolve, reject) => {
				const script = document.createElement('script');
				script.onload = () => resolve();
				script.onerror = (e) => reject(e);
				script.src = urls[handle];
				document.body.appendChild(script);
			});
		});
	}

	function loadExtra(handle, pos) {
		const count = (scriptExtras[handle] && scriptExtras[handle][pos]) || 0;
		let promise = Promise.resolve();

		for (let i = 0; i < count; i++) {
			promise = promise.then(() => runExtraScript(handle, pos, i));
		}

		return promise;
	}

	function loadWPScript(handle) {
		return loadExtra(handle, 'before')
			.then(() => loadExternalScript(handle))
			.then(() => loadExtra(handle, 'after'));
