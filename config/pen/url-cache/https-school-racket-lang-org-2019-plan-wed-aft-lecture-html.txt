the <span class="RktSym">pfsh</span> that we&rsquo;ve
implemented and the <span class="RktSym">pfsh</span> that we want is that we have to put
<span class="RktSym">run</span> before every program name. Instead of <span class="RktPn">(</span><span class="RktSym">run</span><span class="stt"> </span><span class="RktSym">ls</span><span class="RktPn">)</span>,
we want to write <span class="RktPn">(</span><span class="RktSym">ls</span><span class="RktPn">)</span>.</p><p>Since macros can do any kind of work at compile time, you might
imagine changing <span class="RktSym">pfsh</span> so that it scans the filesystem and
builds up a set of definitions based on the programs that are
currently available via the <span class="stt">PATH</span> environment variable. That&rsquo;s not
how scripting languages are meant to work, though. Also, it&rsquo;s likely
to cause trouble to use the filesystem and environment-variable state
at such a fine granularity to determine bindings of a module.</p><p>Instead, we would like to change the default meaning of parentheses.
In Racket, a pair of parentheses mean a function call by default. In
<span class="RktSym">pfsh</span>, a pair of parentheses should mean running an external
program by default. The &ldquo;by default&rdquo; part concedes that an
identifier after an open parenthesis can change the meaning of the
parenthesis, such as when <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> appears after an open
parenthesis. Otherwise, though, it&rsquo;s as if a <span class="RktSym">function-call</span>
identifier appears after the open parenthesis to specify a
function-call form... and <span class="RktSym">function-call</span> exists, except that
it&rsquo;s spelled <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span>.</p><p><div class="SIntrapara">In other words, in the <span class="RktSym">racket</span> language, when you write
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><p><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span><span class="hspace">&nbsp;</span><span class="RktVal">1</span><span class="hspace">&nbsp;</span><span class="RktVal">2</span><span class="RktPn">)</span></p></blockquote></div><div class="SIntrapara">since <span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span> is not bound as a macro or core syntactic form,
that expands to
</div><div class="SIntrapara"><blockquote class="SCodeFlow"><p><span class="RktPn">(</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span><span class="hspace">&nbsp;</span><span class="RktSym"><a href="http://docs.racket-lang.org/reference/generic-numbers.html#%28def._%28%28quote._~23~25kernel%29._%2B%29%29" class="RktValLink" data-pltdoc="x">+</a></span><span class="hspace">&nbsp;</span><span class="RktVal">1</span><span class="hspace">&nbsp;</span><span class="RktVal">2</span><span class="RktPn">)</span></p></blockquote></div><div class="SIntrapara">The <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> provided by <a href="http://docs.racket-lang.org/reference/index.html" class="RktModLink" data-pltdoc="x"><span class="RktSym">racket</span></a> is defined as a
macro that expands to the core syntactic form for function calls. That
core form is also called <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> internally, but in the rare
case that we have to refer to the core form, we use the alias
<span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25plain-app%29%29" class="RktStxLink" data-pltdoc="x">#%plain-app</a></span>.</div></p><p><div class="SIntrapara">To change the default meaning of parentheses for <span class="RktSym">pfsh</span>, then,
we can rename <span class="RktSym">pfsh:run</span> to <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> on export:

<span class="RktSym"><a href="http://docs.racket-lang.org/reference/module.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25module-begin%29%29" class="RktStxLink" data-pltdoc="x">#%module-begin</a></span> and <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span>, so you may wonder how many

<span class="RktSym"><a href="http://docs.racket-lang.org/reference/__top.html#%28form._%28%28quote._~23~25kernel%29._~23~25top%29%29" class="RktStxLink" data-pltdoc="x">#%top</a></span>, and <span class="RktSym"><a href="http://docs.racket-lang.org/reference/__top-interaction.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25top-interaction%29%29" class="RktStxLink" data-pltdoc="x">#%top-interaction</a></span>.</p><h5>11.2.1<tt>&nbsp;</tt><a name="(part.___datum)"></a><span class="RktSym"><a href="http://docs.racket-lang.org/reference/quote.html#%28form._%28%28quote._~23~25kernel%29._~23~25datum%29%29" class="RktStxLink" data-pltdoc="x">#%datum</a></span></h5><p><div class="SIntrapara">Try including a number in a <span class="stt">"pfsh7.rkt"</span> program like this:

It&rsquo;s also easy to change <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> to match and distinguish
function and stream shapes&#8212;<wbr></wbr>but what should a function definition
expand to? We have defined <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> so that it treats its
&ldquo;function&rdquo; position as a string name of an external program. When an
open parenthesis is followed by the name of a function that we have
defined within the script, then we&rsquo;d like the application form to mean
a function call, instead.</p><p>Here are two ways to make the adaptation work:</p><ul><li><p>We can change <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> so that it inspects an identifier
in the &ldquo;function&rdquo; position to check whether the identifier is
bound. If so, the <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> corresponds to a function
call.</p><p>In this case, a <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> for a function can expand to a
regular <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span>.</p></li><li><p>We can make <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> bind a function name as a macro, so
that using the name after an open parenthesis triggers a
function-specific macro instead of the generic <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span>
form.</p><p>In this case, a <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define%29%29" class="RktStxLink" data-pltdoc="x">define</a></span> for a function needs to expand
to <span class="RktSym"><a href="http://docs.racket-lang.org/reference/define.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._define-syntax%29%29" class="RktStxLink" data-pltdoc="x">define-syntax</a></span> to bind the function name as a
macro.</p></li></ul><p>Slightly different behaviors fall out from each of these strategies.
With the first strategy, an identifier that is bound to a string for a
program name cannot be used to run the program, because using the
identifier after an open parenthesis would trigger a function call.
With the second strategy, a name bound to a string still works as a
program name, but a function identifier doesn&rsquo;t work as an argument to
another function (unless we do a little more work to make that
possible). Both approaches are viable, and either could be made to fit
a preferred behavior, so let&rsquo;s try both of them.</p><h5>11.3.1<tt>&nbsp;</tt><a name="(part._.Detecting_.Bindings)"></a>Detecting Bindings</h5><p>To try the first strategy, we need <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> to recognize whether
an identifier has a binding or not. Since the <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> macro
receives only an immediate application form, how can it know what
definitions are in the rest of the module? That is, although the
<span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span> macro can do any work its wants at compile time, it
doesn&rsquo;t have a handle on the whole module to inspect it. The macro
expander itself must know about bindings, because it uses binding
information to determine which macro should handle an expansion.
Happily for our <span class="RktSym"><a href="http://docs.racket-lang.org/reference/application.html#%28form._%28%28lib._racket%2Fprivate%2Fbase..rkt%29._~23~25app%29%29" class="RktStxLink" data-pltdoc="x">#%app</a></span>, the macro expander shares its binding
information with macros in several ways, including through a
<span class="RktSym"><a href="http://docs.racket-lang.org/reference/stxcmp.html#%28def._%28%28quote._~23~25kernel%29._identifier-binding%29%29" class="RktValLink" data-pltdoc="x">identifier-binding</a></span> function.</p><p>The <span class="RktSym"><a href="http://docs.racket-lang.org/reference/stxcmp.html#%28def._%28%28quote._~23~25kernel%29._identifier-binding%29%29" class="RktValLink" data-pltdoc="x">identifier-binding</a></span> function takes an identifier and
reports <span class="RktVal">#f</span> if the identifier has no binding. Otherwise, it
reports some information about the binding, such as which module
(possibly the current one) contains a definition of the identifier.
For our purposes, we do not care about the additional details, so we
can just check whether <span class="RktSym"><a href="http://docs.racket-lang.org/reference/stxcmp.html#%28def._%28%28quote._~23~25kernel%29._identifier-binding%29%29" class="RktValLink" data-pltdoc="x">identifier-binding</a></span> returns
<span class="RktVal">#f</span>.</p><p><div class="SIntrapara">Specifically, we add a new clause to the <span class="RktSym"><a href="http://docs.racket-lang.org/syntax/Parsing_Syntax.html#%28form._%28%28lib._syntax%2Fparse..rkt%29._syntax-parse%29%29" class="RktStxLink" data-pltdoc="x">syntax-parse</a></span> form in
