<p>I wanted to change the colours of Anzu's counter, taking the foreground from the Isearch match background and the background from the fringes, like this</p>
<p><a href="https://i.stack.imgur.com/N2eoc.png" rel="nofollow noreferrer"><img src="https://i.stack.imgur.com/N2eoc.png" alt="enter image description here" /></a></p>
<p>So I evaluated</p>
<pre><code>(face-spec-set 'anzu-mode-line `((t :foreground ,(face-attribute 'isearch :background)
                                    :background ,(face-attribute 'fringe :background)))
               'face-defface-spec)
</code></pre>
<p>and it worked. I added it to my <code>init.el</code>, restarted the emacs server... and then it looked like this:</p>
<p><a href="https://i.stack.imgur.com/Mlqg3.png" rel="nofollow noreferrer"><img src="https://i.stack.imgur.com/Mlqg3.png" alt="enter image description here" /></a></p>
<p>How do I fix that?</p>
<hr />
<p>PS If I launch Emacs from the command line instead of opening a client frame, then the faces get the right colours. It also works if I put strings in place of the <code>face-attribute</code> forms:</p>
<pre><code>(face-spec-set 'anzu-mode-line '((t :foreground &quot;magenta3&quot;
                                    :background &quot;grey95&quot;))
               'face-defface-spec)
</code></pre>

    <div class="s-prose js-post-body" itemprop="text">
<p>Update: Emacs 27 provides <code>server-after-make-frame-hook</code>, which makes it easier to copy face properties for client sessions.</p>
<pre><code>(defun my-anzu-faces ()
  (face-spec-set 'anzu-mode-line `((t
                                    :foreground ,(face-attribute 'isearch :background)
                                    :background ,(face-attribute 'mode-line-inactive :background)))
                 'face-defface-spec))
(if (daemonp)
    (add-hook 'server-after-make-frame-hook #'my-anzu-faces)
  (my-anzu-faces))
</code></pre>
<p>This works for client sessions as well as stand-alone ones.</p>
<hr />
<p>Well, it seemed that the problem was that the face properties which I wanted <code>anzu-mode-line</code> to take were still unspecified at the time of setting it, so I kept on messing around with hooks and <code>with-eval-after-load</code> until I came up with this</p>
<pre><code>(add-hook 'after-make-frame-functions
          (lambda (frame)
            (with-selected-frame frame
              (face-spec-set 'anzu-mode-line `((t
                                                :foreground ,(face-attribute 'isearch :background)
                                                :background ,(face-attribute 'fringe :background)))
                             'face-defface-spec))))
</code></pre>
<p>As you can probably guess, I'm not sure it is the optimal solution, but it is a solution.</p>
