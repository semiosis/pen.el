<p>A <b>fifth-generation programming language</b> (<b>5GL</b>) is any <a href="/wiki/Programming_language" title="Programming language">programming language</a> based on problem-solving using constraints given to the program, rather than using an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> written by a programmer.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> Most <a href="/wiki/Constraint_programming" title="Constraint programming">constraint-based</a> and <a href="/wiki/Logic_programming" title="Logic programming">logic programming</a> languages and some other <a href="/wiki/Declarative_language" class="mw-redirect" title="Declarative language">declarative languages</a> are fifth-generation languages.
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>While <a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">fourth-generation programming languages</a> are designed to build specific programs, fifth-generation languages are designed to make the computer solve a given problem without the programmer. This way, the user only needs to worry about what problems need to be solved and what conditions need to be met, without worrying about how to implement a routine or algorithm to solve them. Fifth-generation languages are used mainly in <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a> research. <a href="/wiki/OPS5" title="OPS5">OPS5</a> and <a href="/wiki/Mercury_(programming_language)" title="Mercury (programming language)">Mercury</a> are examples of fifth-generation languages.<sup id="cite_ref-bala_2-0" class="reference"><a href="#cite_note-bala-2">&#91;2&#93;</a></sup> as is <a href="/wiki/ICAD_(software)" title="ICAD (software)">ICAD</a>, which was built upon <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>.  <a href="/wiki/KL-ONE" title="KL-ONE">KL-ONE</a> is an example of a related idea, a <a href="/wiki/Frame_language" class="mw-redirect" title="Frame language">frame language</a>.
</p><p>In the 1980s, fifth-generation languages were considered to be the way of the future, and some predicted that they would replace procedural programming with constraint based programming for all tasks that could be framed as a series of logical constraints.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> Most notably, from 1982 to 1993, <a href="/wiki/Japan" title="Japan">Japan</a><sup id="cite_ref-Ref1_4-0" class="reference"><a href="#cite_note-Ref1-4">&#91;4&#93;</a></sup><sup id="cite_ref-Ref2_5-0" class="reference"><a href="#cite_note-Ref2-5">&#91;5&#93;</a></sup> put much research and money into their <a href="/wiki/Fifth_generation_computer" class="mw-redirect" title="Fifth generation computer">fifth-generation computer systems project</a>, hoping to design a massive computer network of machines using these tools.
</p><p>However, as larger programs were built, the flaws of the approach became more apparent. It turns out that, given a set of constraints defining a particular problem, deriving an efficient algorithm to solve it is a very difficult problem in itself. This crucial step cannot yet be automated and still requires the insight of a human programmer.
</p>
<h2><span class="mw-headline" id="Common_misconception">Common misconception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=2" title="Edit section: Common misconception">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Vendors have been known on occasion to advertise their languages as 5GL. Most of the time they actually sell <a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">4GLs</a> with a higher level of automation and <a href="/wiki/Knowledge_base" title="Knowledge base">knowledge base</a>. Because the hype of the 1980s faded away and the projects were eventually all dropped, 5GL awareness has also dropped; this has opened doors to the vendors to re-use the term in marketing their new tools, without causing much controversy among the current generations of programmers.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup>
</p>
