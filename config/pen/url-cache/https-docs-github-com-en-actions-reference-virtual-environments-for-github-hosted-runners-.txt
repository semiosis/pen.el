<p>A GitHub-hosted runner is a virtual machine hosted by GitHub with the GitHub Actions runner application installed. GitHub offers runners with Linux, Windows, and macOS operating systems.</p>
<p>When you use a GitHub-hosted runner, machine maintenance and upgrades are taken care of for you. You can run workflows directly on the virtual machine or in a Docker container.</p>
<p>You can specify the runner type for each job in a workflow. Each job in a workflow executes in a fresh instance of the virtual machine. All steps in the job execute in the same instance of the virtual machine, allowing the actions in that job to share information using the filesystem.</p>
<p>The GitHub Actions runner application is open source. You can contribute and file issues in the <a href="https://github.com/actions/runner">runner</a> repository.</p>
<h3 id="cloud-hosts-for-github-hosted-runners"><a aria-hidden="" tabindex="-1" class="doctocat-link" href="#cloud-hosts-for-github-hosted-runners"><svg aria-hidden="" role="img" class="octicon-link" viewBox="0 0 16 16" width="16" height="16" fill="currentColor" style="display:inline-block;user-select:none;vertical-align:middle"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Cloud hosts for GitHub-hosted runners</h3>
<p>GitHub hosts Linux and Windows runners on Standard_DS2_v2 virtual machines in Microsoft Azure with the GitHub Actions runner application installed. The GitHub-hosted runner application is a fork of the Azure Pipelines Agent. Inbound ICMP packets are blocked for all Azure virtual machines, so ping or traceroute commands might not work. For more information about the Standard_DS2_v2 machine resources, see "<a href="https://docs.microsoft.com/azure/virtual-machines/dv2-dsv2-series#dsv2-series">Dv2 and DSv2-series</a>" in the Microsoft Azure documentation.</p>
<p>GitHub hosts macOS runners in GitHub's own macOS Cloud.</p>
<h3 id="workflow-continuity-for-github-hosted-runners"><a aria-hidden="" tabindex="-1" class="doctocat-link" href="#workflow-continuity-for-github-hosted-runners"><svg aria-hidden="" role="img" class="octicon-link" viewBox="0 0 16 16" width="16" height="16" fill="currentColor" style="display:inline-block;user-select:none;vertical-align:middle"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Workflow continuity for GitHub-hosted runners</h3>
<p>If GitHub Actions services are temporarily unavailable, then a workflow run is discarded if it has not been queued within 30 minutes of being triggered. For example, if a workflow is triggered and the GitHub Actions services are unavailable for 31 minutes or longer, then the workflow run will not be processed.</p>
<p>In addition, if the workflow run has been successfully queued, but has not been processed by a GitHub-hosted runner within 45 minutes, then the queued workflow run is discarded.</p>

<p>The Linux and macOS virtual machines both run using passwordless <code>sudo</code>. When you need to execute commands or install tools that require more privileges than the current user, you can use <code>sudo</code> without needing to provide a password. For more information, see the "<a href="https://www.sudo.ws/man/1.8.27/sudo.man.html">Sudo Manual</a>."</p>
<p>Windows virtual machines are configured to run as administrators with User Account Control (UAC) disabled. For more information, see "<a href="https://docs.microsoft.com/windows/security/identity-protection/user-account-control/how-user-account-control-works">How User Account Control works</a>" in the Windows documentation.</p>

<div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-accent-emphasis color-bg-accent f5">
<p><strong>Note:</strong> If you use an IP address allow list for your GitHub organization or enterprise account, you cannot use GitHub-hosted runners and must instead use self-hosted runners. For more information, see "<a href="/en/actions/hosting-your-own-runners/about-self-hosted-runners">About self-hosted runners</a>."</p>

<p>To get a list of IP address ranges that GitHub Actions uses for GitHub-hosted runners, you can use the GitHub REST API. For more information, see the <code>actions</code> key in the response of the "<a href="/en/rest/reference/meta#get-github-meta-information">Get GitHub meta information</a>" endpoint.</p>
<p>Windows and Ubuntu runners are hosted in Azure and subsequently have the same IP address ranges as the Azure datacenters. macOS runners are hosted in GitHub's own macOS cloud.</p>
<p>Since there are so many IP address ranges for GitHub-hosted runners, we do not recommend that you use these as allow-lists for your internal resources.</p>
<p>The list of GitHub Actions IP addresses returned by the API is updated once a week. </p>

<p>GitHub executes actions and shell commands in specific directories on the virtual machine. The file paths on virtual machines are not static. Use the environment variables GitHub provides to construct file paths for the <code>home</code>, <code>workspace</code>, and <code>workflow</code> directories.</p>

<p>Actions that run in Docker containers have static directories under the <code>/github</code> path. However, we strongly recommend using the default environment variables to construct file paths in Docker containers.</p>
<p>GitHub reserves the <code>/github</code> path prefix and creates three directories for actions.</p>
<ul>
<li><code>/github/home</code></li>
