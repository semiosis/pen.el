<p>Converts a wide character to its narrow multibyte representation.
</p>
<div class="t-li1"><span class="t-li">1)</span> If <code>s</code> is not a null pointer, the function determines the number of bytes necessary to store the multibyte character representation of <code>wc</code> (including any shift sequences, and taking into account the current multibyte conversion state <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span>ps</span></span>), and stores the multibyte character representation in the character array whose first element is pointed to by <code>s</code>,  updating <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span>ps</span></span> as necessary. At most <span class="t-c"><span class="mw-geshi c source-c">MB_CUR_MAX</span></span> bytes can be written by this function.</div>
<div class="t-li1"><span class="t-li"></span> If <code>s</code> is a null pointer, the call is equivalent to <span class="t-c"><span class="mw-geshi c source-c">wcrtomb<span class="br0">&#40;</span>buf, L<span class="st0">'<span class="es5">\0</span>'</span>, ps<span class="br0">&#41;</span></span></span> for some internal buffer <code>buf</code>.</div>
<div class="t-li1"><span class="t-li"></span> If wc is the null wide character <span class="t-c"><span class="mw-geshi c source-c">L<span class="st0">'<span class="es5">\0</span>'</span></span></span>, a null byte is stored, preceded by any shift sequence necessary to restore the initial shift state and the conversion state parameter <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span>ps</span></span> is updated to represent the initial shift state.</div>
<div class="t-li1"><span class="t-li"></span> If the environment macro <span class="t-c"><span class="mw-geshi c source-c">__STDC_ISO_10646__</span></span> is defined, the values of type <span class="t-c"><span class="mw-geshi c source-c"><span class="kw4">wchar_t</span></span></span> are the same as the short identifiers of the characters in the Unicode required set (typically UTF-32 encoding); otherwise, it is implementation-defined. In any case, the multibyte character encoding used by this function is specified by the currently active C locale.</div>
<div class="t-li1"><span class="t-li">2)</span> Same as <span class="t-v">(1)</span>, except that</div>
<div class="t-li1"><span class="t-li"></span> if <code>s</code> is a null pointer, the call is equivalent to <span class="t-c"><span class="mw-geshi c source-c">wcrtomb_s<span class="br0">&#40;</span><span class="sy3">&amp;</span>retval, buf, <span class="kw4">sizeof</span> buf, L<span class="st0">'<span class="es5">\0</span>'</span>, ps<span class="br0">&#41;</span></span></span> with internal variables <code>retval</code> and <code>buf</code> (whose size is greater than <span class="t-lc">MB_CUR_MAX</span>)</div>
<div class="t-li1"><span class="t-li"></span> the result is returned in the out-parameter <code>retval</code></div>
<div class="t-li1"><span class="t-li"></span> the following errors are detected at runtime and call the currently installed <a href="/w/c/error/set_constraint_handler_s" title="c/error/set constraint handler s">constraint handler</a> function:
<dl><dd><ul><li> <code>retval</code> or <code>ps</code> is a null pointer.
</li><li> <code>ssz</code> is zero or greater than <span class="t-lc">RSIZE_MAX</span> (unless <code>s</code> is null)
</li><li> <code>ssz</code> is less than the number of bytes that would be written (unless <code>s</code> is null)
</li><li> <code>s</code> is a null pointer but <code>ssz</code> is not zero 
</li></ul>
</dd><dd>As with all bounds-checked functions, <code>wcrtomb_s</code>  only guaranteed to be available if <span class="t-c"><span class="mw-geshi c source-c">__STDC_LIB_EXT1__</span></span> is defined by the implementation and if the user defines <span class="t-c"><span class="mw-geshi c source-c">__STDC_WANT_LIB_EXT1__</span></span> to the integer constant <span class="t-c"><span class="mw-geshi c source-c"><span class="nu0">1</span></span></span> before including <a href="/w/c/string/wide" title="c/string/wide"><tt>&lt;wchar.h&gt;</tt></a>.</div>
</dd></dl>

<h3><span class="editsection">[<a href="/mwiki/index.php?title=c/string/multibyte/wcrtomb&amp;action=edit&amp;section=2" title="Edit section: Return value">edit</a>]</span> <span class="mw-headline" id="Return_value">Return value</span></h3>
<div class="t-li1"><span class="t-li">1)</span> On success, returns the number of bytes (including any shift sequences) written to the character array whose first element is pointed to by <code>s</code>. </div>
<div class="t-li1"><span class="t-li"></span> On failure (if <span class="t-c"><span class="mw-geshi c source-c">wc</span></span> is not a valid wide character), returns <span class="t-c"><span class="mw-geshi c source-c"><span class="br0">&#40;</span><a href="http://en.cppreference.com/w/c/types/size_t"><span class="kw100">size_t</span></a><span class="br0">&#41;</span><span class="sy2">-</span><span class="nu0">1</span></span></span>, stores <span class="t-lc"><a href="/w/c/error/errno_macros" title="c/error/errno macros">EILSEQ</a></span> in <span class="t-lc"><a href="/w/c/error/errno" title="c/error/errno">errno</a></span>, and leaves <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span>ps</span></span> in unspecified state.</div>
<div class="t-li1"><span class="t-li">2)</span> Returns zero on success and non-zero on failure, in which case, <span class="t-c"><span class="mw-geshi c source-c">s<span class="br0">&#91;</span><span class="nu0">0</span><span class="br0">&#93;</span></span></span> is set to <span class="t-c"><span class="mw-geshi c source-c"><span class="st0">'<span class="es5">\0</span>'</span></span></span> (unless <code>s</code> is null or <code>ssz</code> is zero or greater than <span class="t-lc">RSIZE_MAX</span>) and <span class="t-c"><span class="mw-geshi c source-c"><span class="sy2">*</span>retval</span></span> is set to <span class="t-c"><span class="mw-geshi c source-c"><span class="br0">&#40;</span><a href="http://en.cppreference.com/w/c/types/size_t"><span class="kw100">size_t</span></a><span class="br0">&#41;</span><span class="sy2">-</span><span class="nu0">1</span></span></span> (unless <code>retval</code> is null)</div>
