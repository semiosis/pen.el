<h2 id="sec:main"><a id="sec:A.16"><span class="sec-nr">A.16</span> <span class="sec-title">library(main): 
Provide entry point for scripts</span></a></h2>





<p>

</p>

<dl class="tags">
<dt class="tag">See also</dt><dd>XPCE users should have a look at <code>library(pce_main)</code>, which 
starts the GUI and processes events until all windows have gone.</dd>
</dl>





<p>
This library is intended for supporting PrologScript on Unix using 
the
<code>#!</code> magic sequence for scripts using commandline options. 
The entry point <a href="/verrijktkoninkrijk/swish/pldoc/man?predicate=main/0">main/0</a> calls 
the user-supplied predicate <span class="pred-ext">main/1</span> passing 
a list of commandline options. Below is `echo' in Prolog (adjust 
/usr/bin/swipl to where SWI-Prolog is installed)
</p>

<pre class="code">
#!/usr/bin/env swipl

:- initialization(main, main).

main(Argv) :-
        echo(Argv).

echo([]) :- nl.
echo([Last]) :- !,
        write(Last), nl.
echo([H|T]) :-
        write(H), write(' '),
        echo(T).</pre>





<dl class="latex">
<dt class="pubdef"><a id="main/0"><strong>main</strong></a></dt><dd class="defbody">Call <span class="pred-ext">main/1</span> using the passed command-line 
arguments. Before calling
<span class="pred-ext">main/1</span> this predicate installs a signal 
handler for <code>SIGINT</code> (Control-C) that terminates the process 
with status 1.</dd>
<dt class="pubdef"><span class="pred-tag">[det]</span><a id="argv_options/3"><strong>argv_options</strong>(<var>+Argv, 
-RestArgv, -Options</var>)</a></dt><dd class="defbody">Generic transformation of long commandline arguments to options. Each 
--Name=Value is mapped to Name(Value). Each plain name is mapped to 
Name(true), unless Name starts with <code>no-</code>, in which case the 
option is mapped to Name(false). Numeric option values are mapped to 
Prolog numbers.



<dl class="tags">
<dt class="tag">See also</dt><dd><code>library(optparse)</code> provides a more involved option library, 
providing both short and long options, help and error handling. This 
predicate is more for quick-and-dirty scripts.</dd>
</dl>


</dd>
</dl>
