For macOS and Windows users, you should now see the Espanso icon on the status bar:</p><p><img alt="Espanso icon on Windows" src="/assets/images/tray_explain_image_windows-a4482a39604313a2484a7361cacf93f8.png">
<img alt="Espanso icon on macOS" src="/assets/images/icon_explain_image_macos-79769f4465a5acf312cd2baa1742c443.png"></p><p>Linux users can check if Espanso is running by opening a Terminal and running:</p><div class="codeBlockContainer_K1bP theme-code-block"><div class="codeBlockContent_hGly"><pre tabindex="0" class="prism-code language-undefined codeBlock_23N8 thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_39YC"><span class="token-line" style="color:#393A34"><span class="token plain">espanso status</span><br></span></code></pre><button type="button" aria-label="Copy code to clipboard" class="copyButton_Ue-o clean-btn">Copy</button></div></div><p>At this point, you are ready to use Espanso! type <code>:espanso</code>, you should see <code>Hi there!</code> appear. If you don&#x27;t see it, please read the troubleshooting
section below.</p><div class="admonition admonition-info alert alert--info"><div class="admonition-heading"><h5><span class="admonition-icon"><svg xmlns="http://www.w3.org/2000/svg" width="14" height="16" viewBox="0 0 14 16"><path fill-rule="evenodd" d="M7 2.3c3.14 0 5.7 2.56 5.7 5.7s-2.56 5.7-5.7 5.7A5.71 5.71 0 0 1 1.3 8c0-3.14 2.56-5.7 5.7-5.7zM7 1C3.14 1 0 4.14 0 8s3.14 7 7 7 7-3.14 7-7-3.14-7-7-7zm1 3H6v5h2V4zm0 6H6v2h2v-2z"></path></svg></span>Troubleshooting</h5></div><div class="admonition-content"><p>If you don&#x27;t see the Espanso icon on macOS or Windows, or if <code>espanso status</code> returned <code>espanso is not running</code>
on Linux, it means Espanso is not currently running on your machine.</p><p>Firstly, try starting Espanso again:</p><ul><li><strong>For Windows users</strong>: click Espanso from the Start Menu, or click <code>START_ESPANSO.bat</code> for
Portable mode users.</li><li><strong>For macOS users</strong>: click on the Espanso app again. If you see a security warning, please follow the steps

These keywords, known as <em>triggers</em>, are defined as simple strings. For example, <code>:date</code>, <code>e&#x27;</code>,
<code>signature</code> and <code>&gt;&gt;up</code> are all valid triggers.</p><p><img alt="Espanso replaces a trigger with some other text" src="/assets/images/new_match_1-e979325325211ce47eadfc789b980651.png"></p><p>The <em>rule</em> that associates a trigger with the replaced text is called <strong>Match</strong> and is a core concept of Espanso.

<code>default.yml</code> and <code>base.yml</code> respectively.
Each of them serves a specific purpose:</p><ul><li><p><strong>The files contained in the <code>match</code> directory define <em>WHAT</em> Espanso should do.</strong>
In other words, this is where you should specify all the custom snippets and actions (aka Matches).
The <code>match/base.yml</code> file is where you might want to start adding your matches, as shown in the
following sections. As the number of snippets grows, you might want to <em>split</em> your matches
over multiple files to make it easier to manage. For example, you might create the <code>match/emails.yml</code>
file with the snippets you use while writing emails. You can learn all about matches in the
<a href="/docs/matches/basics/">Matches section</a>.</p></li><li><p><strong>The files contained in the <code>config</code> directory define <em>HOW</em> Espanso should perform its expansions.</strong>
In other words, this is were you should specify all Espanso&#x27;s parameters and options.
The <code>config/default.yml</code> file defines the options that will be applied to <em>all applications by default</em>,
unless an <em>app-specific configuration</em> is present for the current app.
For example, you might want to enable emoji snippets for all apps in the <code>config/default.yml</code> file,
but disable them when using Slack in the <code>config/slack.yml</code> file.
You can learn all about configurations in the <a href="/docs/configuration/basics/">Configuration section</a>.</p></li></ul><p>All these files are defined using the widely popular <a href="https://en.wikipedia.org/wiki/YAML" target="_blank" rel="noopener noreferrer">YAML</a> format.</p><h2 class="anchor anchorWithStickyNavbar_31ik" id="creating-your-own-matches">Creating your own Matches<a class="hash-link" href="#creating-your-own-matches" title="Direct link to heading">‚Äã</a></h2><p>That&#x27;s enough theory for now, let&#x27;s start with some action!
