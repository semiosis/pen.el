identity</a></li></ul></div></nav></aside><main class="col col--7"><article><header><h1 class="title_f1Hy">Creating a new Racket language</h1><div class="container_mt6G margin-vert--md"><time datetime="2023-02-25T15:50:00.000Z">February 25, 2023</time> · <!-- -->2 min read</div></header><div id="__blog-post-container" class="markdown"><p>If you&#x27;re interested in creating new programming languages or domain specific languages and also in scheme or LISP you will definitively encounter Racket at some point. Which is described on the official website as &quot;the Language-Oriented Programming Language&quot; so it is something to look into it.</p>
<p>Learning a new language comes with his own struggles, such us having an IDE of some sort and support for your preferred OS and not always there is a good solution at least not for novices. If you are into Emacs you probably don&#x27;t need an IDE to code in a LISP language, but if you&#x27;re a .NET developer like me used to Visual Studio or Rider working with Emacs might not be the easiest path to learn a new language.</p>
<p>Racket comes with a cross-platform IDE which I run with no particular issues on Windows and on Fedora Linux and there are extensions and a language service to work with VS Code. Overall an easy setup with an IDE of some sort that allows a developer to start tinkering without much hassle.</p>
<p>Racket comes with a lot of documentation and there is a lot of interests in his language building capabilities and people much more competent than me on both Racket and on building programming languages wrote about it. However I struggled to find a minimal setup of a project to start a new programming language or DSL.</p>
<p>That&#x27;s why, to my own benefit I created a repo with a very basic setup of a new programming language with a custom surface syntax. <a href="https://github.com/davidelettieri/sample-racket-language" target="_blank" rel="noopener noreferrer">Here</a> is the repo.</p>
<p>I might update the repo in the future to have nice error messages and a more complex syntax for the language but for now it is a decent starting point to start building your own language.</p></div><footer class="docusaurus-mt-lg"><div class="row margin-top--sm theme-blog-footer-edit-meta-row"><div class="col"><b>Tags:</b><ul class="tags_jXut padding--none margin-left--sm"><li
